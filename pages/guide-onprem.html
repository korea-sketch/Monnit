SQL Server에서 Super Admin(sa) 사용자 활성화

Enterprise

iMonnit Enterprise DB 관리 시 sa 사용자 사용 가능(기본 비활성). 설치 중 DB 연결 완료에 필요할 수 있음.

sa 활성화: SSMS 실행·로그인 → Object Explorer에서 서버 확장 → Security > Logins → sa 우클릭 > Properties > Status → Login 아래 Enabled 선택 → OK. 비밀번호 설정: sa Properties > General 탭에서 비밀번호 입력·확인 → OK. 서버 재시작: SQL Server 우클릭 > Restart. 이후 Enterprise Configuration Utility 첫 단계에서 DB 자격증명 확인 가능.

최종 수정일: 2023년 11월 1일

SQL Server에서 Mixed Mode 인증 활성화

Enterprise

iMonnit Enterprise는 SQL Server가 Mixed Mode 인증으로 작동해야 합니다.

절차: SSMS 실행·로그인 → Object Explorer에서 SQL Server 우클릭 > Properties > Security → Server Authentication에서 SQL Server and Windows Authentication mode 라디오 선택 → OK → 자동 재시작 안 되면 SQL Server 우클릭 > Restart. 이후 Windows·SQL 인증 모두 지원.

Windows에서 Enterprise DB SQL 연결 테스트

Enterprise

Enterprise 설치 프로그램 밖에서 자격증명을 확인하기 위해 UDL 파일로 DB 연결 테스트 가능.

UDL 파일 생성: 바탕화면 우클릭 > New > Text Document → 탐색기 Tools > Folder Options > View에서 "Hide file extensions" 해제 → 파일을 "dbtest.udl"로 이름·확장자 변경(경고 무시).

연결 테스트: dbtest.udl 우클릭 > Properties > Provider 탭에서 Microsoft OLE DB for SQL Server 선택 → Next > Connection 탭 → SQL 서버 호스트명\인스턴스명 입력 → Windows 인증이면 "Use Windows NT Integrated security", SQL 인증이면 "Use a specific user name and password" 선택 → DB 선택 → Test Connection. 경로·자격증명 정확성 확인용(연결 실패 시 권한 문제 또는 Mixed Mode 필요 가능성).

게이트웨이를 iMonnit Express/Enterprise로 Pointing

Enterprise

Express/Enterprise 사용 시 게이트웨이를 호스팅 Windows PC의 호스트/IP로 Point해야 함(Default Interface 통신을 iMonnit Cloud 대신 Express/Enterprise로). Ethernet Gateway와 USB WSA만 Express 호환.

Ethernet Gateway Pointing (먼저 게이트웨이 Unlock 필요, outbound 3000 포트로 Unlock 명령 수신):

  • 온라인(iMonnit.com): 계정에 게이트웨이 추가·활성 확인 → iMonnit.com/sethost → 게이트웨이 ID·SC Code 입력 → Server Settings → 호스트 주소·포트(보통 PC IP·3000) 입력 → Save.
  • 오프라인(로컬 HTTP): 게이트웨이 IP(DHCP) 또는 AutoIP(169.254.100.1) 접속 → Settings > Default Server → Server Address를 sensorsgateway.com에서 PC IP로 변경 → Save Changes → 재부팅.

USB Gateway(WSA) Pointing: WSA Unlock 상태는 호스팅 PC에 연결됨(PC 변경 시 재Unlock). localhost:5000 접속 → Active·Online 확인 → support@monnit.com에 Gateway ID·Unlock Key 발송하여 Unlock 요청 → Unlock 후 Host Name을 sensorsgateway.com에서 PC IP/주소로 변경 → Save.

최종 수정일: 2025년 2월 7일

게이트웨이가 갑자기 Enterprise와 통신 중단 시

Enterprise

Enterprise 측 점검:

  • 버전: 4.0+ 지원(3.5.0.0+ 기본 지원 가능, 이전은 미지원). 업그레이드는 구매 후 12개월 내 무료(이후 20% 유지보수료).
  • Enterprise Wireless Gateway Server 서비스 실행 확인(inbound 3000 포트로 데이터 수신·DB 전달, 미실행 시 모든 게이트웨이가 동시 통신 중단).
  • 장치가 Enterprise DB에 추가됐는지 확인(오프라인은 XML 다운로드).
  • 설치 만료 배너: Trial(60일)·연 50센서·영구 250+센서 라이선스. 만료 시 데이터 수신 중단·"This installation is expired" 배너 → 유효 키로 갱신.
  • 센서 한도: 한도 도달 시 서비스는 실행되나 수신 거부(ExceptionLog에 예외). 포트 80 테스트 성공·3000 실패면 한도 초과 가능. "Only EnterpriseBasic sensors allowed" 오류 참조.
  • DB 연결·크기: SQL Express 무료판은 10GB 한도.
  • Communication Check Utility: 서버에서·동일 네트워크 노드 PC에서 테스트(포트 보안·네트워크 구성 진단).

게이트웨이 측 점검(Ethernet Gateway 4 기준): 하단 LED(로컬 이더넷, 녹색=유효 IP, 빨강=IP 없음), 두 번째 LED(Enterprise 서버 연결, 녹색=연결). 게이트웨이 IP ping → Local HTTP Configuration Page 접속 → "Default Server Name/IP"가 서버 주소인지 확인.

iMonnit Enterprise 설치 체크리스트

Enterprise

Enterprise 사이트 로드 오류 시(또는 설치 전) 확인:

  1. Configuration Utility를 관리자 권한으로 실행.
  2. Enterprise Wireless Gateway Server 서비스 실행 확인.
  3. SQL에서 Mixed Mode 인증 활성화.
  4. SQL 데이터정렬(collation)을 SQL_Latin1_General_CP1_CI_AS로 설정(대소문자 구분 — 아니면 "Must declare the scalar variable @Rv" 오류).
  5. Windows Features 설치: .NET Framework 3.5(2.0·3.0 포함), ASP, ASP.NET 3.5·4.8, .NET Extensibility 3.5·4.8, Static Content, .NET 3.5·4.8 HTTP Activation(주로 IIS > World Wide Web Services > Application Development Features).
  6. Windows Default Website 중지 또는 호스팅 적절히 구성(캐시·쿠키 리셋).
  7. sa 사용자 사용 시 활성화·비밀번호 재설정(미사용 시 db_datareader·db_datawriter 권한 필요, 설치는 sysadmin 역할 확인됨).
  8. 디스크 암호화(FIPS 포함) 비활성화.
  9. Active Directory 비활성화.
  10. 브라우저 캐시 리셋(중요).

iMonnit Enterprise 완전 제거 방법

Enterprise

완전 재설치 시에만 수행:

  1. 서비스 중지: Services에서 Enterprise Gateway Server > Stop.
  2. 서비스 삭제: 관리자 cmd에서 `sc delete "Enterprise Wireless Gateway Server"` → [SC] DeleteService SUCCESS.
  3. DB 제거: SSMS에서 Enterprise database 우클릭 > Delete(이름 모르면 C:\inetpub\wwwroot\Enterprise\의 web.config 연결 문자열 참조), "close existing connections" 체크.
  4. IIS 웹사이트 제거: IIS Manager > Sites > Enterprise Portal 중지·Remove → Application Pools > Enterprise Portal_AppPool 중지·Remove → Default Web Site도 제거.
  5. 폴더 삭제: C:\inetpub\wwwroot\의 Enterprise 폴더, %appdata%의 Enterprise 폴더.
  6. Setup Utility 제거: Add or remove programs에서 Enterprise Setup Utility > Uninstall(C:\Program Files (x86)\Enterprise Configuration Utility 폴더 수동 삭제).

최종 수정일: 2024년 5월 28일

Enterprise Wireless Gateway Server 서비스 실행 확인

Enterprise

이 서비스는 게이트웨이 수신 통신을 받아 구성 포트(기본 3000)로 데이터 전달. 게이트웨이가 데이터를 보내려면 실행 필수.

확인: Enterprise 서버에서 Services.msc 실행 → Enterprise Wireless Gateway Server 찾기 → status "running" 확인 → "stopped"면 우클릭 > Start Service.

미실행 시 증상: 모든 게이트웨이·센서가 통신 불가. 흔한 원인: Windows 업데이트, 비호환 Windows(Home·Server 지원, Pro·Enterprise는 Active Directory 비활성화 시), 시작 권한, 보안 소프트웨어, 포트 충돌(inbound TCP/UDP 3000, HTTP/HTTPS 80/443), Activation Key 문제, 특수문자 포함 SQL 비밀번호, SQL 사용자 읽기/쓰기 권한 부족, 설치 체크리스트 미충족.

Enterprise

Legacy Enterprise - 장치 추가 시 "could not be transferred to a new network"

Enterprise

버전 3.6.2.1 이하에서 발생. 원인: (1) 인터넷 미연결로 메타데이터 자동 조회 불가, (2) 장치가 이미 다른 네트워크에 추가됨, (3) TLS 1.2 업그레이드로 조회 요청 미수신.

  • 이미 추가됨: Administration > Reports > Device ID로 확인("Not Found" 또는 Account/Network 표시).
  • TLS 1.2: http://downloads.imonnit.com/misc/Footer.zip 의 Footer.ascx를 C:\inetpub\wwwroot\Enterprise\Views\Shared\에 교체.

Monnit 외부의 센서 프로필별 구성 값

Sensors

각 센서 프로필은 고유한 구성 값·속성을 가지며 센서 동작·판독값에 영향. MINE·Serial MODBUS 게이트웨이로 수정 가능하나, Monnit은 이러한 설정 구성에 필요한 설명을 명시적으로 제공하지 않습니다(특히 Advanced Vibration·CO2 등 복잡한 프로필).

MINE의 센서 구성: 모든 센서 유형은 "SensorEdit" 메서드를 가지며, 매개변수는 iMonnit의 해당 센서 Edit 페이지 매개변수와 동일. 매개변수는 "Nullable"이라 변경할 값만 전달 가능(예: Heartbeat만 전달하면 나머지는 기존 값 유지, "dirty" 플래그도 자동 처리). 일반적으로 iMonnit UI에서 구성 가능한 값이면 MINE에서 같은 값을 사용하면 동일한 결과.

적용되지 않는 경우: 일부 복잡한 프로필은 raw Sensor 객체를 MINE에서 수정할 수 있으나, 센서 동작을 방해하는 값 전달을 막는 guard가 없습니다(MINE raw Sensor 편집·Serial MODBUS 게이트웨이 구성 모두 해당). 이러한 구성은 내부 정보가 필요하므로 공개되지 않으며, 잘못 적용 시 센서가 작동 불능이 될 수 있습니다.
