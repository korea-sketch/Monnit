01 · 고객 프로필

서버실·금고·보관실 온도·누수·출입을 관리해야 하는 은행·금융기관·지점 운영팀.

금융 지점은 서버실·통신 장비실·금고·보관실 환경을 안정 관리해야 합니다. 온도 상승·누수·도어 개방·습도 변화를 늦게 잡으면 업무 중단·자산 리스크로 이어지므로, 주요 데이터를 자동 기록해 지점 시설 리스크를 통합 관리합니다.

02 · 현재의 문제점

금융 지점의 서버실과 보관 공간은 환경 이상과 출입 이벤트를 빠르게 확인해야 합니다.

  • 서버실 환경 리스크 — 온도 상승·습도 변화가 통신·서버 장비 장애 유발
  • 누수 사고 위험 — 기계실·배관 누수가 업무 중단·장비 손상
  • 출입 확인 부족 — 금고·보관실 도어 이벤트 지연 시 보안 리스크
  • 지점별 관리 편차 — 다지점 상태 분산 시 본사 통합 확인난

03 · Monnit 솔루션

금융 지점 온도·습도·누수·도어를 모니터링하고 iMonnit에서 지점별 알림·이력 제공.

온도·습도·누수 감지·개폐 감지·움직임 감지 센서를 지점 핵심 지점에 적용해 iMonnit에서 봅니다. 이탈 시 알려 서버실 장애를 예방하고 다지점 시설을 본사가 통합 관제합니다.

구성 센서: 온도 · 습도 · 누수 감지 · 개폐 감지 · 움직임 감지

04 · 제안 단계

장애 시 피해 큰 지점 전산실·ATM 코너 우선 → 전 지점망 단계 확장

  • PHASE 1 중점 공간 지정 (1일) — 지점 서버실·통신 장비실·금고를 우선 지정합니다.
  • PHASE 2 센서 설치 (1~2일) — 온습도·누수·도어·움직임 센서를 핵심 지점에 설치합니다.
  • PHASE 3 본사 통합 설정 (2~3일) — 지점별 통합 대시보드와 알림 라우팅을 설정합니다.
  • PHASE 4 전 지점 확장 (1~2개월) — 검증된 구성을 전 지점망으로 확장합니다.
