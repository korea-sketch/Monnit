센서의 하드웨어 오류(Hardware Failure)

코인셀 배터리로 동작하는 센서의 경우, 이 측정값은 일반적으로 배터리가 잘못 장착되어 발생합니다. 배터리를 하우징 안으로 더 밀어 넣어야 하며, 배터리 위치가 센서의 데이터 측정 능력을 방해하고 있는 것입니다. 배터리가 내부 회로 기판 가장자리의 원형 포트에 닿아 있을 가능성이 높습니다.

이쑤시개, 종이클립 등 가느다란 도구로 배터리 제거 구멍을 통해 센서에서 배터리를 밀어냅니다.

센서의 전력 커패시터가 완전히 방전될 수 있도록 60초간 기다립니다.

배터리를 제거할 때 사용한 도구(이쑤시개, 종이클립 등)로 배터리를 센서 안으로 끝까지 다시 넣습니다.(손가락으로는 충분히 들어가지 않을 수 있습니다.)

소프트웨어를 확인해 센서가 올바른 정보를 표시하는지 확인합니다. 그렇지 않으면 이 과정을 반복합니다. 센서가 계속 이 오류 메시지를 표시하면 센서가 고장났을 수 있습니다. 센서가 아직 보증 기간 내라면 Monnit 고객 지원에 연락해 교체를 조율하십시오.

센서가 AA 배터리로 동작하는 경우 센서가 고장났을 수 있습니다. 배터리를 빼고 60초간 기다린 뒤 다시 넣어 측정값이 바뀌는지 확인합니다. 바뀌지 않으면, 센서가 아직 보증 기간 내라면 Monnit 고객 지원에 연락해 교체를 조율하십시오.

게이트웨이 및 Add-On 기기의 Poll Interval

Poll Interval은 Control Unit, Local Alert 같은 Add-On 기기의 원하는 동작에 중요한 구성입니다. Polling 과정은 Add-On 기기가 원하는 응답 시간으로 명령이나 입력에 응답할 수 있게 합니다.

고려할 Polling 이벤트는 두 가지입니다. 첫째, 게이트웨이가 Monnit 소프트웨어를 폴링하여 명령을 받아야 합니다. 게이트웨이가 명령을 받으면 Add-On 기기가 게이트웨이를 폴링하여 명령을 실행해야 합니다. 따라서 게이트웨이 Settings에 Poll Rate가, Add-On 기기 Settings에 Poll Rate가 있습니다. 이 문서는 폴링 구성을 이해하는 데 필요한 세부 사항을 제공합니다.

게이트웨이 Polling

Add-On 기기에서 명령이 실행되는 과정은 Monnit 소프트웨어에서 명령이 시작되는 것에서 출발합니다. 명령이 Add-On 기기에 도달하려면 게이트웨이가 소프트웨어를 폴링하여 Add-On 기기용 명령을 대기열에 넣어야 합니다. 따라서 게이트웨이는 게이트웨이의 Sensor Network에 있는 기기가 실행할 Add-On 기기 명령이 있는지 확인하는 빈도인 Poll Rate 구성을 제공합니다.

참고: Poll Rate가 0으로 설정되면 게이트웨이는 게이트웨이의 Heartbeat에 Monnit 소프트웨어를 폴링합니다.

참고: 모든 게이트웨이가 Poll Rate 구성을 제공하는 것은 아닙니다. 예를 들어 셀룰러 게이트웨이는 데이터 사용이 고려 사항이므로 일부 셀룰러 게이트웨이는 Settings 탭에 Poll Rate 구성을 제공하지 않습니다.

게이트웨이의 Poll Rate는 게이트웨이의 Settings > Poll Rate에서 확인할 수 있습니다.

Add-On 기기 Poll Interval

Add-On 기기의 Poll Interval은 게이트웨이가 실행하도록 대기열에 넣은 명령이 있는지 확인하기 위해 Add-On 기기가 게이트웨이와 통신하는 빈도입니다. 즉 Add-On 기기가 명령을 실행할 수 있는 최소 시간은 게이트웨이의 Poll Rate + Add-On 기기 Poll Interval + 전송 시간입니다. Add-On 기기가 명령에 응답해야 하는 최소 시간이 있는 경우, Monnit 시스템을 구성할 때 중요한 고려 사항입니다.

참고: 센서의 Poll Interval이 0으로 설정되면 센서는 센서의 Heartbeat에 게이트웨이를 폴링합니다.

Add-On 기기의 Poll Rate는 센서의 Settings > Poll Interval에서 확인할 수 있습니다.

예시

이 예에서는 (기본) 5분 하트비트와 0 Poll Rate로 구성된 Ethernet Gateway 4와, 10분 하트비트 및 1분 Poll Interval로 구성된 Control Unit을 사용합니다. 이 구성에서 제어 명령이 Control Unit에 도달하는 최대 응답 시간은 6분(전송 시간 추가)이고, 최소 시간은 1분 남짓(전송 시간 추가)입니다.

(Control Unit이 마지막으로 오후 5:04:30에 표준 하트비트로 체크인)

오후 5:12:05 - 게이트웨이가 5분 간격 표준 하트비트로 iMonnit Online에 체크인하고 Control Command를 폴링(없음)

오후 5:12:30 - Control Unit이 게이트웨이에 Control Command를 폴링(없음)

오후 5:13:30 - Control Unit이 게이트웨이에 Control Command를 폴링(없음)

오후 5:14:30 - Control Unit이 표준 하트비트로 게이트웨이에 체크인하고 Control Command를 폴링(없음)

오후 5:15:30 - Control Unit이 게이트웨이에 Control Command를 폴링(없음)

오후 5:15:41 - iMonnit Online의 Action이 Relay 1 On 명령을 대기열에 넣음

오후 5:16:30 - Control Unit이 게이트웨이에 Control Command를 폴링(없음, 게이트웨이가 Action 트리거 후 아직 iMonnit Online을 폴링하지 않음)

오후 5:17:05 - 게이트웨이가 5분 간격 표준 하트비트로 iMonnit Online에 체크인하고 Control Command를 폴링(iMonnit Online에서 Relay 1 On 명령을 받아 Control Unit 전달용으로 대기열에 넣음)

오후 5:17:30 - Control Unit이 게이트웨이에 Control Command를 폴링(Relay 1 On 명령 감지) - Control Unit Relay 1이 On으로 설정됨

위 예에서 Action이 오후 5:15:41에 iMonnit Online에서 Control - Relay 1 On 명령을 트리거했고, Control Unit이 오후 5:17:30에 명령을 받았습니다. 게이트웨이의 Poll Rate가 0으로 설정되어 있었으므로 게이트웨이는 하트비트에만 iMonnit Online에서 Control Command를 폴링했습니다. 그런 다음 Control Unit이 1분마다 게이트웨이에 Control Command를 폴링했습니다. 따라서 Control Unit은 게이트웨이가 iMonnit Online에서 Control Command를 받은 하트비트로부터 1분 이내에 Control Command를 받았습니다.

Poll Interval을 이해하는 것은 Control Unit, Local Alert 같은 Add-On 기기의 기대 응답 시간을 얻는 데 핵심입니다. Poll Interval 동작에 관한 추가 문의가 있으면 언제든 Monnit Support에 문의하십시오.

Control Unit의 Default State

Monnit Control Unit에는 각 Relay의 정상 동작 상태를 결정하는 Default State 구성이 있습니다. 각 Relay는 Off 또는 On의 Default State를 갖도록 독립적으로 구성할 수 있습니다.

Default State란?

Default State는 Control Unit의 전원이 켜질 때 각 Relay의 사전 구성된 상태입니다. Default State가 Off로 구성되면 전원이 켜질 때 그 Relay는 off(Relay 열림)입니다. On으로 구성되면 전원이 켜질 때 그 Relay는 on(Relay 닫힘)입니다.

Default State는 Control Unit이 동작 중일 때 실행되나요?

참고: Default State에 관한 흔한 질문은 Relay 명령이 실행된 후 Control Unit이 Default State로 돌아가는지 여부입니다. Default State는 Relay 명령이 Control Unit으로 전송된 후 실행되지 않습니다. Control Unit이 명령을 받으면, Relay 상태를 변경하는 추가 명령이 전송될 때까지 Relay는 그 상태를 유지합니다. 타이머가 있는 Relay 명령의 경우, 타이머가 만료되면 Relay가 반대 상태로 전환되지만 이는 Default State로 결정되는 것이 아닙니다.

Default State는 세 가지 시나리오에서 적용됩니다:

  1. Control Unit의 전원이 처음 켜질 때 Relay가 Default State로 시작합니다.
  2. 구성이 Control Unit으로 전송되어 수락된 후.
  3. 1세대 Control Unit이 있고 페어링된 센서가 Aware 상태가 아닌 경우.

1세대 Control Unit의 페어링 센서 기능이란?

이는 Alta 세대 Control Unit의 기능이 아닙니다. 플랫폼 개선으로 불필요해졌기 때문입니다. Monnit의 1세대 Control Unit에는 Control Unit의 Default State를 페어링 센서와 함께 활용하는 기능이 있습니다.

Control Unit의 Settings에서 Paired Sensor ID 필드에 입력된 센서 ID가 표준 동작 모드이면, 연결된 Relay는 Default State가 됩니다. 페어링 센서가 통신 범위 내에 있고 Aware 상태에 진입하면 Relay가 비-Default 상태로 전환됩니다.

Control Unit의 Default State는 성공적인 Control Unit 구축에 중요한 동작일 수 있습니다. 관련 문의는 언제든 support@monnit.com 으로 연락하십시오.

Control Unit Relay 테스트

Control Unit Relay의 열림/닫힘에 문제가 의심되면 멀티미터로 릴레이를 테스트할 수 있습니다. 이 문서는 그 방법에 관한 정보를 제공합니다.

LED 표시 확인

각 Control Unit에는 물리적 LED 4개(Relay 1, Relay 2, Radio, Power)가 있습니다. Control Unit Relay가 닫히면(회로 완성) LED가 켜집니다. Relay 1 또는 Relay 2 LED가 켜진 것이 보이면 그 릴레이 회로가 현재 닫혀 있는 것입니다. 아래 이미지에서는 Relay 1과 Relay 2 회로가 모두 닫혀 있습니다.

iMonnit Online에서 확인

LED로 Relay 회로가 닫혀 있는지 확실히 알 수 있지만, iMonnit Online 포털에서도 검토할 수 있습니다. 센서가 현재 게이트웨이와 통신 중이면, iMonnit Online에 표시된 상태도 Relay가 On(닫힌 회로)인지 Off(열린 회로)인지 나타냅니다.

리드(leads)의 저항 테스트

Monnit Control Unit은 전압을 공급하지 않지만, 회로가 닫히면 리드의 저항을 테스트할 수 있습니다.

리드를 테스트하려면 각 릴레이의 리드 사이에 멀티미터를 연결하고 저항 값을 확인합니다.

  • Relay가 On(닫힌 회로)이면 10A Control Unit은 100mΩ, 30A Control Unit은 50mΩ의 저항이 보입니다.
  • Relay가 Off(열린 회로)이면 ∞(무한대) 저항이 보입니다.

릴레이가 켜지거나 꺼지면 그에 따라 저항 변화가 보여야 합니다.

각 릴레이의 리드 사이 저항을 테스트한 후 저항 값이 변하지 않으면 릴레이 회로가 손상되었을 수 있습니다. 이 경우 support@monnit.com 으로 도움을 요청하십시오.
