Poll Interval(폴링 간격) 이해

Configurations

Poll Interval은 ALTA Control Unit·Local Alert 같은 add-on 장치가 원하는 응답 시간으로 명령에 반응하게 하는 필수 구성입니다. 두 가지 폴링 이벤트가 있습니다: (1) 게이트웨이가 Monnit 소프트웨어를 폴링해 명령을 받음, (2) add-on 장치가 게이트웨이를 폴링해 실행.

게이트웨이 폴링: 명령은 Monnit 소프트웨어에서 시작됨. 게이트웨이의 Poll Rate는 게이트웨이가 add-on 장치 명령 실행 여부를 확인하는 빈도(Settings > Poll Rate). Poll Rate가 0이면 게이트웨이 Heartbeat에 폴링. (셀룰러 게이트웨이는 데이터 사용량 고려로 Poll Rate 미제공일 수 있음.)

Add-on 장치 Poll Interval: add-on 장치가 게이트웨이에 큐잉된 명령을 확인하는 빈도(Settings > Poll Interval). 명령 실행 최소 시간 = 게이트웨이 Poll Rate + 장치 Poll Interval + 전송 시간. 장치 Poll Interval이 0이면 장치 Heartbeat에 폴링.

예시 (Ethernet Gateway 4: 5분 Heartbeat·0 Poll Rate, Control Unit: 10분 Heartbeat·1분 Poll Interval): 최대 응답 시간 6분(+전송), 최소 1분 남짓(+전송). iMonnit 규칙이 5:15:41 PM에 Relay 1 On 명령을 큐잉 → 게이트웨이가 5:17:05 PM Heartbeat에 iMonnit 폴링해 명령 수신 → Control Unit이 5:17:30 PM 폴링에 명령 감지·Relay 1 On.

최종 수정일: 2025년 3월 28일

ALTA 무선 Control Unit 작동 방식

Control Unit

ALTA 무선 10A·30A Control Unit은 회로를 자동으로 열고 닫는 두 개의 릴레이 스위치를 제공합니다. 릴레이는 전원을 공급하지 않고 회로를 완성/개방하며, 전원 회로에 연결되면 릴레이가 닫힐 때 장치를 작동시킵니다.

기본 릴레이 작동 순서: (1) iMonnit에서 Action 트리거 또는 수동 명령으로 릴레이 작동/해제 명령 발생 → (2) 게이트웨이가 iMonnit을 폴링해 명령 큐잉(Heartbeat 또는 Poll Rate에 수신) → (3) Control Unit이 Heartbeat/Poll Rate에 게이트웨이 폴링 → (4) 큐잉된 명령을 처리해 릴레이 활성화/비활성화.

릴레이 On/Off 명령 방법:

  • Action(규칙): Sensor Reading, Aware State, Schedule 등으로 트리거. 조건 정상 복귀 시 Auto-Acknowledge 활성화 시 Disarm/Rearm. 단, Rearm 후 릴레이가 자동으로 이전 상태로 돌아가지 않음. Disarm되지 않으면 Snooze 시간에 최대 24시간 명령 재발송. 이전 상태로 되돌리려면 별도 Action 생성 필요.
  • 수동 명령: iMonnit의 Control 탭에서 큐잉 순서대로 실행.

Toggle: 트리거 시 릴레이를 반대 상태로 전환(On→Off, Off→On). Timer: 릴레이를 일정 시간 On/Off 후 반대 상태로 복귀.

폴링: 응답 속도는 게이트웨이·Control Unit Poll Interval 합으로 결정(예: 게이트웨이 5분 + Control Unit 1분 = 최대 6분).

Command Queue: Control 탭의 Control Unit Commands 큐에 명령 표시. Clear History 버튼으로 대기 명령 삭제(대역폭 문제 해결 시 유용).

LED 4개(녹색): Relay 1(작동 시 점등), Relay 2(작동 시 점등), Radio(게이트웨이 연결 시 점등), Power(전원·작동 시 점등). Radio LED가 꺼지면 센서처럼 연결 문제 해결.

문제 해결: 게이트웨이가 같은 네트워크인지 확인, Clear History, Poll Rate 구성, 게이트웨이·Control Unit 전원 재시작, Radio LED 점등 확인, 릴레이 저항 확인.

최종 수정일: 2025년 2월 10일

Control Unit 릴레이 테스트

Control Unit

릴레이 개폐 문제가 의심되면 멀티미터로 테스트할 수 있습니다.

LED 확인: Control Unit에는 Relay 1, Relay 2, Radio, Power 4개 LED. 릴레이가 닫히면(회로 완성) LED 점등.

iMonnit Online 확인: 게이트웨이와 통신 중이면 iMonnit 상태가 릴레이 On(닫힘)/Off(열림) 표시.

리드 저항 테스트: Control Unit은 전압을 공급하지 않으나 회로가 닫히면 저항 측정 가능. 각 릴레이 리드 사이에 멀티미터 연결:

  • 릴레이 On(닫힘): 10A는 100 mΩ, 30A는 50 mΩ 저항.
  • 릴레이 Off(열림): ∞(무한) 저항.

저항 값이 변하지 않으면 릴레이 회로 손상 가능성이 있으니 support@monnit.com 문의.

Control Unit의 Default State(기본 상태) 이해

Control Unit

Control Unit은 각 릴레이의 정상 작동 상태를 결정하는 Default State 구성을 가집니다(각 릴레이를 Off 또는 On으로 독립 구성).

Default State란? Control Unit 전원 켜질 때 각 릴레이의 사전 구성 상태(Off면 릴레이 열림, On이면 릴레이 닫힘).

Action 명령 후 Rearm되면 Default State로 돌아가나? 아니요. 명령 수신 시 릴레이는 추가 명령이 올 때까지 그 상태 유지. (타이머 명령은 타이머 만료 후 반대 상태로 전환되나 이는 Default State와 무관.)

Default State 적용 4가지 경우: (1) Control Unit 최초 전원 켜질 때, (2) 게이트웨이에 합류할 때, (3) 구성이 전송·수락된 후, (4) Generation 1 Control Unit에서 페어링 센서가 Aware State가 아닐 때.

예기치 않게 Default State로 전환되는 경우: 불안정한 전원(전원 조건 해결), 전송 범위 내 여러 게이트웨이 간 전환(네트워크 분리), 구성 편집(설정 변경 시 Default State로 복귀).

Generation 1의 페어링 센서 기능: ALTA 세대에는 없는 기능. Paired Sensor ID 필드에 입력된 센서가 표준 모드면 릴레이가 Default State, Aware State 진입 시 비-Default State로 전환.

최종 수정일: 2025년 4월 24일

iMonnit 규칙으로 Control Unit 릴레이 On/Off

Control Unit

iMonnit 규칙(Sensor Reading, Aware State, 일정 등으로 트리거)으로 ALTA Control Unit의 두 릴레이를 작동/해제할 수 있습니다.

예시: 온도 센서가 80°F 초과 시 Relay 1로 팬을 켜고, 75°F 미만 시 끄기(릴레이는 팬과 전원 사이 공통선을 연결/차단).

규칙 1 (80°F에 Relay 1 On): Rules → +Create New Rule → Sensor Reading → 온도 센서 선택(Control Unit이 아님) → Condition Greater Than 80°F → Save → Set Up Tasks에서 Command Control Unit → Done adding tasks → + 버튼으로 Relay Command 활성화(녹색) → Send command to turn Relay 1 토글로 원하는 상태 → (Set Command Timer 토글로 특정 시간) → Save → Done adding tasks → 규칙 이름 → Save.

규칙 2 (75°F에 Relay 1 Off): 동일하나 Condition Less Than 75°F, Relay 1 Off 명령.

작동 방식: 센서가 80°F 초과 보고 → 2분 내 릴레이 작동(게이트웨이 1분 + Control Unit 1분 Poll + 전송) → 팬 켜짐. 규칙이 Disarm/Rearm되어도 릴레이는 자동으로 꺼지지 않으므로 별도 규칙(75°F Off)이 필요. 결과: 80°F 초과 시 팬 On(75°F까지 유지), 75°F 미만 시 Off(80°F까지 유지).

최종 수정일: 2025년 3월 28일

iMonnit 예약 규칙으로 Control Unit 릴레이 On/Off

Control Unit

일정 기반으로 릴레이를 작동시킬 수 있습니다(예: 영업시간에 맞춰 보일러 On/Off). Scheduled Time Rule 사용.

System Action 고려: 예약 규칙은 자동 Disarm/Rearm되지 않으므로 수동 또는 System Action 필요.

예약 규칙 생성: 새 규칙 → Scheduled Time → Schedule for에 트리거 시간 → Save → Set Up Tasks에서 Command Control Unit → 대상 Relay 옆 + → (타이머 원하면 Set command timer On·시간 입력) → Save → Create System Action → Full Reset / Target Rule {Target Self} / Delay 2 Minute Delay → Add → Done → I'm done adding tasks → 규칙 이름 → Save → 트리거 센서 추가(어떤 센서든, 추가하지 않으면 규칙 미작동).

예약 규칙이 구성대로 릴레이를 트리거하고 자신을 재설정하여 24시간마다 재실행. (주/월/커스텀 일정은 Schedule 탭(달력 아이콘)에서 추가 가능.)

Serial MODBUS Gateway와 Control Unit 사용

Control Unit

Monnit Thermostat 사용

Thermostat

Monnit Thermostat은 통합 동작 센서를 활용해 재실(occupancy) 상태에 따라 에너지 효율을 최적화하는 HVAC 관리 솔루션입니다. 재실/비재실 상태별 온도 범위를 구성합니다.

Thermostat 연결: 각 핀의 기능을 이미지로 확인(잘못된 연결이 문제의 흔한 원인).

설정 변경 활성화 - Jumper: 설정 잠금 점퍼가 양쪽 핀에 있으면 설정 변경 불가. 구성하려면 점퍼 제거 필요.

Thermostat 로직: 통합 동작 센서가 재실 여부에 따라 온도 설정을 동적으로 조정(자동 트리거 또는 수동으로 occupied/unoccupied 상태 시작 가능).

Dashboard: iMonnit Online에서 Occupied/Unoccupied 설정 조정, 수동 상태 시작, HVAC 팬 제어. (1) 현재 상태의 냉난방 임계값 슬라이드 조정, (2) 숫자 입력 또는 +/-로 양 상태 임계값 조정, (3) Fan Control(Off / 20분 On / 1시간 On / 4시간 On), (4) Change Occupancy State(지정 시간 동안 수동 조정 후 타이머 만료 시 정상 복귀).

LED 상태: "Pwr"(전원)과 "RF"(무선) LED 점등 시 전원·게이트웨이 연결 정상. 난방 등 기능 활성 시 해당 LED 점등.

Thermostat 설정:

  • Name: 고유 장치 이름.
  • Heartbeat Interval(분): Normal 상태에서 게이트웨이와 통신 간격.
  • Unoccupied Settings: 비재실 시 냉난방 On/Off 시점.
  • Temperature Deadpoint: 냉난방이 켜진 후 꺼지는 버퍼(예: 버퍼 5°F, 난방 65°F에 켜지면 70°F에 꺼짐).
  • Turn Heat on at: 이 값 미만이면 난방 켜짐.
  • Turn Cool on at: 이 값 초과면 냉방 켜짐.
  • Occupied Settings: 재실 시 냉난방 On/Off 시점.
  • Set Occupied on Motion for: 재실 상태 트리거(버튼/동작 감지/소프트웨어 버튼) 후 이중 임계값 설정으로 복귀하는 시간.

Advanced Settings:

  • Trigger On Motion: 켜면 움직임에 Aware 신호·서버로 데이터 포인트 강제 전송.
  • Fan Control: Auto(HVAC가 팬 제어), Auto+Periodic(큰 주기 시작 시 일정 시간 팬 작동, Fan On Period·Fan Auto Period와 함께 사용, 냉난방 이벤트 시 Periodic 시간 재설정), Active Fan Control(냉난방 시작/중지 시 Thermostat가 팬 능동 제어, Fan Start Time과 함께 사용).
  • Fan Period: Fan Interval 동안 팬이 강제 On되는 시간(Fan Auto Period보다 작아야 함).
  • Fan Interval: Fan On Period가 작동하는 간격(Fan On Period보다 커야 함).
  • Fan Start/Stop Time for Heater, Fan Start/Stop Delay for Cooler: 난방·냉방 시작/중지 후 팬 작동/정지 시점.

추가 옵션:

  • Scale: 화씨/섭씨 표시.
  • Calibration: Fan Override, Occupancy Override, Unoccupancy Override 교정. Temperature/Humidity Offset: 실제 온도와 보고 온도의 차이를 보정(다른 장치와 정렬).

최종 수정일: 2024년 11월 1일

ALTA MultiStage Thermostat 구성·설치 가이드

Thermostat

개요: ALTA MultiStage Thermostat은 일반·히트펌프·이중연료(dual fuel) HVAC 시스템을 최대 3단 난방·2단 냉방으로 제어합니다. 통합 동작 센서로 재실 자동 감지 후 비재실 시 에너지 절약 상태 진입. 비밀번호 보호된 iMonnit으로만 제어 가능(무단 조정 방지, 본질적 변조 방지), 전면 패널 다색 LED로 모드·상태 표시.

연결: 각 핀 기능을 이미지로 확인(잘못된 연결이 문제 원인). Jumper: 잠금 점퍼가 양쪽 핀에 있으면 설정 변경 불가, 제거해야 구성 가능. 로직: 통합 동작 센서로 재실 여부에 따라 온도 동적 조정(수동 시작도 가능).

iMonnit Dashboard: (1) Dial Indicator(Occupied Setpoint·Unoccupied 냉난방 임계값 표시), (2) Setpoint 조정(숫자/+/-), (3) Fan Control(Off/20분/1시간/4시간), (4) Force Occupancy State(20분/1시간/4시간 강제), (5) Heat/Cool Mode(Heat/Cool/Auto/Off).

LED: Pwr·RF 점등 시 전원·게이트웨이 연결 정상, 기능 활성 시 해당 LED 점등.

주요 설정:

  • System Type: HVAC 시스템 유형·제어 항목 설정(정의하지 않으면 안전 기능으로 HVAC 미작동).
  • Heat/Cool Mode: 난방/냉방/비상난방(emergency heat — 주 난방 우회, 보조 난방만 사용)/냉난방 자동 제어.
  • Temperature Buffer: 냉난방 활성 온도 결정(예: setpoint 70°F·버퍼 5° → 65°F에 난방 On, 70°F에 Off). 정확한 setpoint가 아닌 0.25°C 오버슈트로 쾌적·효율 향상.
  • Occupied Set Point / Occupied Timeout: 재실 냉난방 Off 온도 / 재실 트리거 후 비재실 설정 복귀 시간.
  • Unoccupied Heating/Cooling Set Points: 비재실 냉난방 Off 온도.
  • Heating/Cooling Activation Time: 1단 실행 후 해당 단 활성화까지 시간.
  • Heating/Cooling Activation Threshold: setpoint 초과 시 즉시 해당 단 활성화.
  • Stage Load Balancing: 1·2단 부하 균형(전체 런타임 비슷하게, 2단 이상에만 적용, 용량 비슷할 때만 활성화 권장).
  • Reversing Valve Control: 리버싱 밸브가 냉방/난방에 에너지 공급할지 정의(Undefined면 미작동).
  • Aware when Occupied: 동작 감지 시 Aware 보고(Occupied Timeout이 rearm 역할).
  • Aware On State Change: 상태 변경 시 Aware 표시(상태 변경은 항상 보고되나 Aware면 게이트웨이 즉시 전송).
  • Fan Control Mode: Auto(HVAC 제어) / Auto+Periodic / On(항상) / Active Fan Control / Off.
  • Fan On Period / Fan Interval / Fan Start·Stop Time: (Control Unit Thermostat과 동일 개념.)

Calibration/Controls: Fan Override(일정 시간 팬 On), Occupancy Override(강제 재실), Unoccupied Override(강제 비재실), Temperature/Humidity Offset(판독값 보정).

Thermostat

Thermostat은 재실 상태(Occupied/Unoccupied)에 따라 다른 온도 설정을 가질 수 있습니다(동작 센서 기반 프리셋). 일반적으로 동작 센서에 의존하나 수동 또는 일정으로 트리거 가능.

System Action 고려: 예약 규칙은 자동 Disarm/Rearm되지 않으므로 수동 또는 System Action 필요.

예약 규칙 생성: 새 규칙 → Scheduled Time → Schedule for에 트리거 시간 → Save → Set Up Tasks에서 Control Thermostat → Thermostat 이름 옆 "+" → Mode(Occupied/Unoccupied)와 On 지속 시간 선택 → Save → Create System Action → Full Reset / Target Rule {Target Self} / Delay 2 Minute Delay → Add → Done → I'm done adding tasks → 규칙 이름 → Save.

구성대로 Thermostat을 트리거하고 자신을 재설정하여 24시간마다 재실행. (주/월/커스텀 일정은 Schedule 탭(달력 아이콘)에서 추가 가능.)
