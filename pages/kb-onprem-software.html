인터넷에 접속되지 않은 서버에서 iMonnit Enterprise를 활성화하는 경우

오프라인 활성화

iMonnit Enterprise 소프트웨어는 Enterprise Activation Key를 검증하기 위해 인터넷 연결을 사용합니다. iMonnit Enterprise를 설치하는 서버가 인터넷에 접속할 수 없으면, 서버에 인터넷 연결 없이도 활성화를 완료할 수 있는 오프라인 활성화 절차가 있습니다. 이 경우 ‘Unable to verify against the online database’ 메시지가 보입니다. 활성화를 완료할 Offline Manual Key를 생성하려면 다른 기기에서 인터넷 접속이 필요합니다. 이 문서는 그 방법을 안내합니다.

단계 요약

  • 일반 활성화 절차를 시도합니다.
  • Manual Token을 가져옵니다.
  • Manual Key를 생성합니다.
  • iMonnit Enterprise 설치 프로그램에 Manual Key를 입력해 활성화를 완료합니다.

일반 활성화 절차 시도

Enterprise Configuration Utility를 실행해 Enterprise를 설치합니다. SQL 데이터베이스 자격 증명을 입력한 후 Enter Key를 클릭합니다. Enterprise Activation Key를 입력하고 Activate를 클릭합니다. 설치 프로그램이 인터넷으로 키를 검증하려 시도합니다. 인터넷 연결이 없으면 ‘Unable to verify key against online database. Contact support at support@monnit.com’ 메시지가 보이고 Offline Activation 프롬프트가 나타납니다.

참고: 이전 버전의 Enterprise 설치 프로그램을 실행하는 경우에도 이 메시지가 보일 수 있습니다. 이는 검증 서버에 적용된 TLS 암호화 업그레이드의 결과입니다. 이전 설치 프로그램은 TLS 1.1을 사용했고, 현재 Monnit 검증 서버에는 TLS 1.2가 적용되어 있습니다. 따라서 TLS 1.1로 키를 보내는 버전(3.6.2.1 이하)의 설치 프로그램을 실행 중이면 Enterprise를 오프라인으로 수동 활성화해야 합니다.

Manual Token 가져오기

이 단계에서 Manual Token이 자동으로 생성됩니다. Manual Token 필드의 텍스트를 복사합니다.

Manual Key 생성

https://www.monnit.com/order/productactivation 웹사이트를 방문해 Manual Token 텍스트를 Product Activation 필드에 입력합니다. OK 버튼을 클릭합니다. 그러면 Manual Key가 생성됩니다. 이 텍스트를 복사합니다.

iMonnit Enterprise 설치 프로그램에 Manual Key 입력

복사한 Manual Key 텍스트를 Enterprise 설치 프로그램의 Manual Key 필드에 붙여넣고 Manual 버튼을 클릭합니다. 그러면 설치가 검증되고 설치를 계속할 수 있습니다. Manual Key가 수락되면 .ky 파일을 복사해야 한다는 다음 메시지가 보입니다:

‘you will need to copy the key and put it in the website directory you create after the installation. You will be able to find the key:

C:/inetpub/wwwroot/Enterprise\Enterprise Server\[Key name].ky’

이는 키가 성공적으로 적용되었음을 나타냅니다. 메시지를 확인한 뒤(파일을 복사할 필요는 없음) 설치를 계속하면 됩니다.

오류

오류가 보이면 Monnit Support에 문의해야 할 수 있습니다. https://www.monnit.com/order/productactivation 사이트에서 Activation Failed 메시지가 보이면 키 재설정을 위해 Monnit Support에 문의해야 할 수 있습니다. 오류가 보이면 사용하려는 Activation Key를 반드시 포함하십시오.

오프라인 활성화 절차는 인터넷 연결이 없을 때 iMonnit Enterprise 소프트웨어를 활성화할 수 있는 편리한 방법입니다. 이 절차에 문제가 있으면 언제든 Monnit Support에 문의하십시오.

iMonnit Enterprise 서버에서 Enterprise Wireless Gateway Server 서비스 실행 확인

Enterprise Wireless Gateway Server는 iMonnit Enterprise 서버에서 Windows의 Services.msc 프로그램으로 실행되는 서비스입니다. 이 서버는 게이트웨이가 구성된 포트(기본 포트 3000)로 서버에 데이터를 전송할 때 들어오는 통신을 수신하는 역할을 합니다. 게이트웨이가 iMonnit Enterprise 서버로 데이터를 보내려면 이 서비스가 실행 중이어야 합니다.

단계 요약

  • iMonnit Enterprise 서버에 접근합니다.
  • 적절한 권한이 있는 사용자로 Windows에서 Services.msc 프로그램을 실행합니다(Windows 검색에서 ‘services’를 검색하면 접근 가능).
  • 서비스 목록에서 Enterprise Wireless Gateway Server를 찾습니다.
  • 상태가 ‘running’인지 확인합니다.
  • 상태가 ‘stopped’이면 서비스를 마우스 오른쪽 버튼으로 클릭하고 ‘Start Service’를 선택합니다.

Communication Check Utility로 게이트웨이의 통신이 수신되고 있는지 확인할 수 있습니다. 관련 정보는 해당 문서의 ‘The Enterprise Wireless Gateway Server’ 섹션에서 확인할 수 있습니다.

서비스가 실행되지 않을 때의 증상

Enterprise Wireless Server 서비스가 Enterprise 서버에서 실행되지 않으면, 서버에 추가된 모든 게이트웨이와 센서가 통신하지 못합니다. 모든 기기가 서버로 데이터 전송을 멈췄다면 가장 먼저 이것을 확인하십시오.

서비스 중지의 가능한 원인

서비스 중지의 흔한 원인은 아래와 같습니다.

  • Windows 업데이트
  • 시작 시 실행을 허용하지 않는 권한
  • 보안 소프트웨어

iMonnit Enterprise에 기기 추가 시 ‘could not be transferred to a new network’ 오류가 보이는 경우

‘[SensorID/GatewayID: xxxxxx] could not be transferred to a new network’ 오류는 iMonnit Enterprise 서버 버전 3.6.2.1 이하에 새 기기를 추가하려 할 때 나타날 수 있습니다. 이 오류는 몇 가지 문제 중 하나를 나타낼 수 있습니다. 이 문서는 이 메시지의 가능한 원인과 해결책을 다룹니다.

이 메시지의 가능한 원인

  1. Enterprise 서버가 인터넷에 연결되어 있지 않아 기기 메타데이터를 자동으로 가져올 수 없음.
  2. 기기가 이미 Enterprise 데이터베이스에 추가되어 Enterprise 내 다른 네트워크에 이미 추가되어 있음.
  3. TLS 1.2 암호화 업그레이드로 인해 Enterprise 서버가 기기 메타데이터를 가져오기 위해 보낸 요청이 Monnit의 lookup 서비스에 도달하지 못함.

iMonnit Enterprise를 호스팅하는 서버가 인터넷에 접속할 수 없는 경우

기본적으로 iMonnit Enterprise 데이터베이스에는 기기 ID가 없으므로 기기를 데이터베이스에 추가해야 합니다. Enterprise는 센서·게이트웨이 ID와 코드를 사용해 중앙 온라인 저장소에서 센서·게이트웨이의 필수 메타데이터를 다운로드할 수 있습니다. 이는 웹 응용 프로그램을 호스팅하는 서버가 인터넷에 접속할 수 있는 경우에만 가능합니다. 게이트웨이와 센서를 데이터베이스에 추가하는 방식은, 데이터베이스에 추가된 후 네트워크 간에 이동하는 방식과 동일합니다. ‘Manage’ 화면에서 ‘Add Gateway/Add Sensor’를 선택하면 Gateway ID/Sensor ID와 코드를 입력하라는 메시지가 표시됩니다. 이 값들은 Monnit 게이트웨이와 센서에 부착된 ID 라벨에 있습니다.

iMonnit Enterprise 버전 4 - 기기 수동 추가

Enterprise 버전 4에서 오프라인으로 기기를 수동 추가하려면 이 문서를 참고하십시오.

레거시 Enterprise - 기기 수동 추가

이는 레거시 Enterprise 버전 3.6.2.1 이하에만 해당합니다.

서버 관리자만 접근할 수 있는 영역이 있으며, 여기서 게이트웨이와 센서를 데이터베이스에 수동으로 추가할 수 있습니다. 이 절차는 서버가 중앙 온라인 데이터베이스에서 정보를 다운로드할 수 없을 때만 수행해야 합니다. 기기를 추가할 계정과 네트워크로 이동합니다. ‘Administration’ 영역의 Network Details 아래에 게이트웨이와 센서를 수동 추가하는 링크가 있습니다. 이 영역에서 게이트웨이와 센서를 서로 다른 계정/네트워크 간에 빠르게 이동할 수도 있습니다.

기기를 데이터베이스에 수동으로 추가해야 하면 iMonnit 온라인 포털에서 올바른 메타데이터를 조회할 수 있습니다. 인터넷에 연결된 웹 브라우저로 https://www.imonnit.com/lookup 에 접속합니다. 센서 또는 게이트웨이 ID와 check-code를 양식에 입력하면 필수 메타데이터를 가져옵니다. 이 정보로 ‘Add new Sensor’ 양식을 작성해 데이터베이스에 기기를 채웁니다. 센서가 데이터베이스에 입력되면 사용자는 기기 라벨의 기기 ID와 코드로 계정 간에 기기를 이동할 수 있습니다.

새 게이트웨이를 데이터베이스에 추가하려면:

  • ‘Add New Gateway’를 선택합니다.
  • 게이트웨이 스티커의 게이트웨이 ID를 입력합니다.
  • 게이트웨이 이름을 지정합니다.
  • 올바른 게이트웨이 유형을 선택합니다.
  • 올바른 radio band를 선택합니다(관리 용도 전용).
  • 올바른 APN Firmware Version을 입력합니다(관리 용도 전용).
  • 올바른 Gateway Firmware Version을 입력합니다(관리 용도 전용).
  • MAC Address(필수 아님)는 네트워크에서 조회하면 입력할 수 있습니다.
  • Save를 클릭합니다.

센서를 데이터베이스에 추가하려면:

  • ‘Add New Sensor’를 클릭합니다.
  • 올바른 Sensor Profile을 선택합니다(이 정보를 올바르게 입력하는 것이 동작에 매우 중요).
  • 센서 하단의 센서 ID를 입력합니다.
  • 센서 이름을 지정합니다.
  • 올바른 센서 유형을 선택합니다:

o Commercial / o Commercial (AA) / o Industrial / o Wi-Fi

  • 펌웨어 버전을 입력합니다.
  • 전원 소스를 선택합니다:

o Commercial (CR2032) / o Commercial (AA) / o Line Power / o Voltage / o Industrial (Tadiran) / o Industrial Solar (Ind Rech) / o Custom (AAA) / o Custom (CR123) / o Wi-Fi (AA)

  • Save를 클릭합니다.

기기가 이미 네트워크에 추가되어 있는 경우

이 메시지는 기기가 이미 데이터베이스에 추가되어 현재 다른 네트워크에 있는 경우에도 나타날 수 있습니다. 이를 판단하려면 아래 단계를 따르십시오.

  • 서버 관리자로 로그인합니다.
  • 탐색 바의 Administration 탭을 클릭합니다.
  • 탐색 바의 Reports 탭을 클릭합니다.
  • Device ID 필드에 기기 ID를 입력합니다.
  • Submit을 클릭합니다.

o 기기가 아직 데이터베이스에 추가되지 않았으면 ‘Not Found’ 텍스트가 보입니다.

o 기기가 데이터베이스에 추가되었으면 Account와 Network 필드에 기기가 추가된 위치가 표시됩니다.

TLS 1.2 업그레이드로 인해 Enterprise 서버가 요청을 완료할 수 없는 경우

Enterprise 버전 3.6.2.1 이하는 TLS 1.1 지원으로 개발되었습니다. 이 버전들이 출시된 이후 Monnit은 서버 통신 시 사용하는 암호화를 TLS 1.2로 업그레이드했습니다. Enterprise 3.6.2.1 이하는 TLS 1.1로 통신을 시도하므로 이 요청들이 Monnit 서버에 도달하지 못합니다. 따라서 이 문제를 해결하려면 Enterprise 서버에 패치가 필요할 수 있습니다. 패치를 적용하려면 아래 단계를 따르십시오.

  1. 다음 링크에서 Footer.ascx 파일을 다운로드합니다: http://downloads.imonnit.com/misc/Footer.zip .
  2. 다음 기본 위치의 기존 Footer.ascx 파일을 교체합니다:

C:\inetpub\wwwroot\Enterprise\Views\Shared\

참고: 위 위치는 기본 위치입니다. 서버가 다른 디렉터리에 설치되었으면 해당 위치로 파일을 복사하십시오.

iMonnit Enterprise 3.6.2.1 이하에서 ‘This installation is expired, contact support to update subscription.’가 보이는 경우

iMonnit Enterprise는 Annual 및 Perpetual 라이선스를 모두 제공합니다. Annual 라이선스의 경우 소프트웨어 활성화에 사용한 날짜로부터 1년 후 키가 만료됩니다. 이 경우 설치 상단에 ‘This installation is expired, contact support to update subscription.’ 배너가 보입니다. 이를 해결하려면 새 annual 키를 Enterprise 설치에 적용해야 합니다.

증상

체험 만료를 나타내는 배너 외에도, 계정의 게이트웨이가 더 이상 통신하지 않는 것을 볼 수 있습니다. 이는 키 만료로 Enterprise Wireless Gateway Server 서비스가 중지된 결과입니다. Enterprise Wireless Gateway Server를 실행하려면 소프트웨어를 재활성화해야 합니다.

Enterprise 센서 10개 Trial

센서 10개 Enterprise 라이선스는 갱신할 수 없습니다. Enterprise Trial 키를 60일간 사용한 후 이 메시지가 보이면 정식 라이선스를 구매해야 합니다.

갱신 키 구매

Enterprise Annual License 제품 페이지를 방문해 사이트에서 키를 구매할 수 있습니다. 교체 키를 받으면 아래 단계로 새 키를 입력할 수 있습니다.

  • Enterprise Wireless Configuration Utility를 실행합니다.
  • Update Key 버튼을 선택합니다.
  • Activation Key를 입력합니다.
  • Activate를 클릭합니다.

이렇게 하면 Enterprise Activation Key가 1년 더 갱신됩니다.

센서 250개 perpetual 라이선스에서 이 메시지가 보이는 경우

레거시 버전의 Enterprise(버전 3.6.2.1 이하)를 실행 중이면 perpetual 250 센서 라이선스를 적용했더라도 이 메시지가 보일 수 있습니다. 이를 해결하려면 Enterprise 버전을 최신 버전 4로 업그레이드해야 합니다.

업그레이드가 옵션이 아닌 경우 해결 방법이 있을 수 있습니다. Support가 최선의 지원을 제공하지만, 이 해결이 성공하지 못하면 Support는 최신 버전 업그레이드를 해결책으로 제안할 것입니다.

먼저 Enterprise Activation Key를 support@monnit.com 으로 제공해야 합니다. 일반적으로 Enterprise Configuration Utility를 실행하고 Update Key 버튼을 클릭하면 Activation Key 필드에 자동으로 채워집니다. 없으면 키를 구매한 주문 번호를 제공해야 합니다. Monnit Support가 재활성화를 위해 키를 재설정합니다. 아래 단계를 참고하십시오.

  • Enterprise Configuration Utility를 실행합니다.
  • Update Key 버튼을 클릭합니다.
  • 전체 Activation Key를 복사합니다.
  • 키를 support@monnit.com 으로 이메일 보냅니다.
  • Support가 키를 재설정하고 완료 확인을 제공합니다.
  • Enterprise Configuration Utility를 실행합니다.
  • Update Key를 클릭합니다.
  • Activation Key를 입력합니다.
  • Activate를 클릭하면 키가 업데이트됩니다(오프라인 활성화가 필요할 수 있음).
  • Enterprise Wireless Gateway Server 서비스를 중지했다가 시작합니다.

이렇게 하면 소프트웨어가 재활성화되고 게이트웨이가 다시 통신할 수 있습니다.

성공하지 못한 경우:

인터넷에 연결되지 않은 상태에서 iMonnit Enterprise 서버에 기기 추가

iMonnit Enterprise는 기기를 Enterprise 포털/데이터베이스에 추가할 때 기본적으로 인터넷 연결을 활용해 기기 메타데이터를 자동으로 다운로드합니다. 인터넷 연결이 없으면 기기 추가 시 ‘SensorID: xxxx not authorized to transfer’ 메시지, 404 오류, 또는 ‘Server Error in '/' Application. The remote name could not be resolved: 'www.imonnit.com'’이 보입니다. 기기를 수동으로 추가해야 합니다. 수동 추가 절차는 인터넷을 통해 iMonnit.com 온라인 센서 포털에 기기를 추가한 뒤, 모든 센서 메타데이터를 담은 XML 파일을 다운로드하고, 그 파일을 Enterprise 서버로 전송한 다음 서버에 업로드하는 것입니다. 이 문서는 이 작업을 수행하는 단계를 제공합니다.

참고: 이 오류는 레거시 버전의 Enterprise에서는 다릅니다. 4.0 이전 버전은 다음 문서를 참고하십시오: iMonnit Enterprise - 기기 추가 시 ‘could not be transferred to a new network’.

Sensor Network의 모든 기기 ID를 XML 파일로 다운로드

인터넷 연결이 없을 때 기기를 Enterprise에 추가하려면, iMonnit Online에서 네트워크 내 모든 기기를 담은 XML 파일을 생성한 뒤 그 파일을 Enterprise 포털로 가져오면 됩니다. 이 XML 파일은 선택한 네트워크의 모든 기기 메타데이터를 담으므로, 그 네트워크에 Enterprise에 둘 기기만 있도록 하십시오.

  1. 인터넷이 되는 컴퓨터를 찾습니다.
  2. iMonnit Online(www.imonnit.com)에 로그인합니다.
  3. (선택) 새 네트워크를 생성합니다.
  • 화면 왼쪽의 networks 탭을 선택합니다.
  • 오른쪽 상단의 ‘+ New Network’ 버튼을 선택합니다.
  • 네트워크 이름을 지정합니다.
  • Save를 선택합니다.
  1. 원하는 네트워크에 기기를 추가합니다.
  • 네트워크를 선택합니다.
  • 화면 오른쪽의 ‘Add Device to Network +’ 버튼을 선택합니다.
  • 기기를 추가할 네트워크를 선택합니다.

  • 각 박스에 기기 ID와 코드를 입력합니다.
  • Add Device를 선택합니다.
  • 추가할 각 기기에 대해 이 과정을 반복합니다.
  1. 모든 기기를 원하는 네트워크에 추가한 후 전체 네트워크와 기기의 메타데이터를 다운로드합니다.
  • 네트워크를 클릭합니다.

  1. Done을 선택합니다.

Ethernet 게이트웨이는 외부 인터넷 연결 없이 컴퓨터에서 무선 센서를 로컬로 모니터링하는 데 사용할 수 있습니다. PC 응용 프로그램과 Ethernet 게이트웨이를 사용하려면 게이트웨이와 PC가 같은 네트워크에 연결되어 있는지 확인하고, 게이트웨이가 인터넷을 사용하는 대신 컴퓨터 소프트웨어로 직접 데이터를 보내도록 구성해야 합니다. 이를 위해 다음이 필요합니다:

  • Monnit Ethernet 게이트웨이
  • Monnit 센서
  • 설치된 iMonnit Express 4 소프트웨어
  • 게이트웨이가 Express를 실행하는 PC에 연결할 수 있는 Ethernet 네트워크
  • Gateway Unlock Code

iMonnit Express 설치

Quick Start Guide는 다음 링크의 문서를 참고하십시오: https://monnit.blob.core.windows.net/content/documents/quickstart/AQS-009-iMonnit-Express-Core.pdf

먼저 iMonnit Express 소프트웨어를 설치해야 합니다. iMonnit Express 4는 레거시 버전과 달리 사용자가 웹 브라우저를 통해 소프트웨어와 상호작용합니다. Express는 PC에 로컬로 호스팅되는 웹사이트입니다. 경험은 로컬로 호스팅된 iMonnit Online을 사용하는 것과 유사합니다.

  1. 다음 링크에서 iMonnit Express 4 설치 프로그램을 다운로드합니다: https://www.monnit.com/support/downloads/ (iMonnit™ Express > iMonnit Express™ (Windows Installer)).
  2. iMonnit Express 설치 프로그램을 실행하고 화면 안내를 따릅니다(필요하면 마우스 오른쪽 버튼 클릭 후 Run as Administrator 선택).
  3. !!중요!! 컴퓨터를 재시작합니다.
  4. 바탕화면에서 iMonnit Express 프로그램을 실행합니다(브라우저가 실행되어 localhost 사이트로 이동).
  5. 구매 시 받은 코드로 Express 소프트웨어를 활성화합니다.
  6. 이메일 주소와 고유 비밀번호로 사용자 이름을 만듭니다. 조합을 다시 입력해 iMonnit Express Core에 로그인합니다.
  7. 게이트웨이와 센서를 iMonnit Express에 추가합니다.

게이트웨이를 Unlock하고 iMonnit Express로 Point

iMonnit Express 같은 로컬 버전의 iMonnit 소프트웨어와 Monnit 게이트웨이를 사용하려면 게이트웨이를 Unlock Code로 Unlock해야 합니다. Unlock되면 게이트웨이를 로컬 iMonnit Express 소프트웨어로 데이터를 보내는 데 사용할 수 있습니다. 이 과정에 관한 정보는 다음 문서를 참고하십시오: 게이트웨이를 사용자 지정 호스트/IP로 Unlock 및 Point(Enterprise/Express/MINE).

참고: 게이트웨이를 이미 Unlock된 상태로 구매하지 않았다면, Unlock을 위해 게이트웨이가 인터넷을 통해 iMonnit Online 포털에 연결해야 합니다.

게이트웨이를 iMonnit Express 소프트웨어에 연결

게이트웨이가 iMonnit Express 소프트웨어로 데이터를 보내려면, 로컬 네트워크에서 TCP 포트 3000을 통해 iMonnit Express를 실행하는 PC와 통신해야 합니다. 이 통신을 허용하도록 네트워크 방화벽이나 PC를 구성해야 할 수 있습니다. 이를 한 후, 게이트웨이가 Express를 실행하는 PC로 데이터를 보내도록 구성할 수 있습니다. 이는 다음 문서에서 참조하는 HTTP Interface(로컬 구성 페이지)로 수행할 수 있습니다: Ethernet Gateway 4 - 로컬 구성 페이지(HTTP Interface) 로컬 접근.

게이트웨이가 (위에서 언급한 대로) Unlock되어 있으면, Data Interfaces > Default Server Configuration 탭에서 iMonnit Express를 실행하는 PC의 IP 주소를 입력하고 Save를 클릭합니다. 게이트웨이가 재부팅되고 iMonnit Express로 데이터를 보내기 시작합니다.

iMonnit Express는 데이터를 로컬에 호스팅해야 하고 iMonnit Online보다 잦은 하트비트가 필요한 사용자에게 좋은 옵션입니다. 추가 문의가 있으면 언제든 Monnit Support에 문의하십시오.

업그레이드 전 iMonnit Enterprise 백업 생성

새 버전으로 업그레이드하기 전에 iMonnit Enterprise 설치를 백업하는 것이 중요합니다. 이 문서는 그 단계를 안내하는 자료를 제공합니다.

단계 요약

  1. 사이트 파일 백업을 생성합니다.
  2. Enterprise 데이터베이스 백업을 생성합니다.
  3. Activation Key를 반드시 기록해 둡니다.

사이트 파일 백업

사이트 파일의 .zip 파일을 생성합니다.

  • C:\inetpub\wwwroot\ 로 이동합니다.
  • Enterprise 폴더를 마우스 오른쪽 버튼으로 클릭합니다.
  • Send To > Compressed (zipped) Folder를 선택합니다.
  • .zip 파일을 안전한 위치에 저장합니다.

SQL 데이터베이스 백업

SQL 데이터베이스의 백업을 생성합니다.

  • SQL Server Management Studio를 실행합니다.
  • 왼쪽 열의 Object Explorer에서 Server를 펼칩니다.
  • Databases를 펼칩니다.
  • Enterprise 데이터베이스를 찾습니다.
  • 데이터베이스를 마우스 오른쪽 버튼으로 클릭하고 Tasks > Backup을 선택합니다.
  • 대상(destination)을 확인합니다.
  • OK를 클릭합니다.

데이터베이스가 백업되며 필요하면 나중에 복원할 수 있습니다.

%appdata% 폴더 백업

설치용 구성 파일이 %appdata% 폴더에 있습니다. 이 폴더의 백업을 생성합니다.

  • Windows 버튼을 선택해 검색에 접근합니다.
  • ‘%appdata%’를 입력하고 Return을 누릅니다.
  • 이 디렉터리에서 ‘Enterprise’ 폴더를 복사합니다(마우스 오른쪽 버튼 클릭 후 Send To > Compressed (zipped) folder로 .zip 파일을 생성할 수 있음).

Activation Key 기록

키 구매 시 Monnit이 보낸 이메일에서 Activation Key를 가져옵니다. 원본 이메일이 없으면 다음 단계로 Enterprise Configuration Utility에서 가져올 수 있습니다.

  • Enterprise Configuration Utility를 실행합니다.
  • Update Key 버튼을 클릭합니다.
  • 현재 키가 Activation Key 필드에 표시될 수 있습니다.

키를 가져올 수 없으면 주문 번호로 Monnit Support에서 가져올 수 있습니다. 원본 주문 번호를 찾아 support@monnit.com 으로 이메일을 보내면 키를 제공받을 수 있습니다.

iMonnit Enterprise SQL 데이터베이스의 유지보수 요구 사항

시간이 지나면 iMonnit Enterprise SQL 데이터베이스에는 유지보수가 필요합니다. 그 유지보수의 정확한 적용은 일반적으로 데이터베이스가 어떻게 사용되는지에 따라 다릅니다. 한 데이터베이스에 최적인 것이 다른 데이터베이스에는 가장 최적이 아닐 수 있습니다. 다만 데이터베이스 유지에 할 수 있는 일반적인 조언이 있습니다. 일반적으로 더 많은 데이터를 담은 테이블에 데이터베이스 호출이 도달할 때 쿼리 성능 문제가 발생하기 시작합니다. 그런 경우의 해결책은 테이블의 데이터 양을 줄이거나, 쿼리가 적시에 실행되기 쉽도록 데이터를 구조화하는 것입니다. 이는 인덱스를 추가하거나 테이블의 통계를 업데이트하여 할 수 있습니다.

데이터 제거

데이터를 정리·제거하는 것이 옵션일 수 있습니다. 예를 들어 고객이 과거 Webhook 데이터가 필요 없다고 판단하면, webhook 쿼리를 더 빠르게 하기 위해 그 데이터를 제거할 수 있습니다. 이를 하려면 데이터를 삭제하거나 테이블을 truncate(테이블의 모든 데이터 제거)해야 합니다. 테이블의 데이터를 삭제하면 delete 문이 끝날 때까지 그 테이블이 차단되며, 외래 키 제약이 있는 동안 다른 테이블에서 참조되는 데이터는 삭제할 수 없다는 점을 유의하십시오. 또한 truncate 문은 즉시 실행되지만, 그 데이터가 다른 테이블에서 명시적으로 참조되지 않더라도 외래 키 제약이 있는 동안에는 수행할 수 없습니다. 데이터의 상당 부분을 제거하면 그 테이블의 통계를 업데이트해야 할 수 있습니다.

인덱싱 및 통계 업데이트

데이터 삭제가 최선이 아닌 경우, 테이블의 하나 이상의 열에 인덱스를 추가하는 것을 선택할 수 있습니다. 인덱스는 선택한 특정 열의 데이터를 구조화·정렬하여 쿼리 속도를 높입니다. 인덱스를 생성하면 더 많은 디스크 공간을 차지합니다. 정확한 크기는 인덱싱하는 열의 데이터 유형과 테이블의 행 수에 따라 다릅니다. 또한 인덱스는 향후 유지보수가 필요할 수 있습니다. 정기적인 간격으로 인덱스를 재구축해야 할 수 있고, 가끔 그 인덱스의 통계를 업데이트해야 할 수도 있습니다. 다만 인덱스는 데이터베이스에 도달하는 쿼리를 최적화하는 강력한 도구입니다. 인덱스를 구축하는 동안에는 테이블이 차단된다는 점을 유의하십시오.

데이터베이스 유지의 정확한 방법은 개별 요구에 따라 다릅니다. 단일 옵션이 모든 경우에 옳은 것은 아닙니다. 어느 시점에는 데이터베이스 관리자를 고용해 직접 사례를 평가하게 하는 것이 최선일 수 있습니다. 위 옵션을 시도하려면 먼저 데이터베이스를 백업하는 것이 좋습니다.

단계 요약

  1. Enterprise Wireless Gateway Server 서비스를 중지합니다.
  • Windows에서 Services.msc 프로그램을 실행합니다(보통 Windows 검색에서 ‘services’ 검색). 참고: 서비스를 중지·시작할 권한이 있어야 합니다.
  • 목록에서 Enterprise Wireless Gateway Server 서비스를 찾습니다.
  • Enterprise Wireless Gateway Server 서비스를 마우스 오른쪽 버튼으로 클릭합니다.

  • iMonnit Enterprise 사이트를 호스팅하는 서버에서 탐색기로 다음 디렉터리로 이동합니다: C:\inetpub\wwwroot\Enterprise\ .

‘Pwd=’는 SQL 사용자의 비밀번호입니다.

  • 메모장으로 이 파일 내용을 편집합니다. 참고: 관리자 권한이 필요합니다.

<connectionStrings> <add name="SqlServer" connectionString="Server=DESKTOP\SQLEXPRESS;Database=Enterprise;Uid=sa;Pwd=password;" /> </connectionStrings>

  • 기존 자격 증명을 업데이트된 자격 증명으로 교체합니다.
  • 같은 디렉터리 C:\inetpub\wwwroot\Enterprise\Enterprise Server\ 에 파일을 저장합니다.
  1. IIS에서 Enterprise Portal 웹사이트를 시작합니다.
  • 이 가이드의 2단계를 참고하여 Actions 사이드바에서 Start를 선택합니다.
  1. Enterprise Wireless Gateway Server 서비스를 시작합니다.
  • 이 가이드의 1단계를 참고하여 팝업 메뉴에서 Start를 선택합니다.

게이트웨이가 iMonnit Enterprise와 예기치 않게 통신을 멈추는 경우

모든 게이트웨이가 갑자기 iMonnit Enterprise Portal로 데이터 전송을 멈추면, 이 문서의 정보가 문제 식별에 도움이 됩니다.

iMonnit Enterprise Portal 검토

가장 먼저 검토할 구성 요소는 iMonnit Enterprise Portal입니다. 이 단계로 서버가 게이트웨이 통신을 수신하고 있는지 확인합니다.

버전 확인

Monnit은 현재 iMonnit Enterprise 버전 4.0 이상을 지원합니다. 레거시 iMonnit Enterprise 3.5.0.0 이상에 대한 기본 지원은 가능할 수 있습니다. 3.5.0.0 이전 버전은 더 이상 지원되지 않습니다. 버전 3.5.0.0 이상에서는 업그레이드가 가능하지만, 3.5.0.0 이전 버전으로의 업그레이드 지원은 중단되었습니다.

iMonnit Enterprise 업그레이드는 Enterprise 라이선스 구매 후 12개월 이내에 포함됩니다. 키가 12개월 미만이면 지원 버전에서 Enterprise 4.0을 추가 비용 없이 다운로드·설치할 수 있습니다. Enterprise 버전 키가 12개월을 넘으면 새 라이선스에 20% 유지보수 비용이 부과됩니다. 지원 요청 시 iMonnit Enterprise 키나 판매 주문 번호를 제공할 준비를 하십시오.

Enterprise Wireless Gateway Server 서비스 실행 확인

이 문서에 따라 Enterprise Wireless Gateway Server 서비스가 실행 중인지 확인하십시오. 이 서비스는 들어오는 포트 3000(기본값이지만 구성 중 변경되었을 수 있음)을 통해 네트워크에서 서버로 데이터를 받아 iMonnit Enterprise 데이터베이스로 전달·처리하는 프로세스입니다. 게이트웨이가 iMonnit Enterprise 데이터베이스로 데이터를 전달하려면 이 서비스가 실행 중이어야 합니다. 서비스가 실행되지 않을 때 가장 흔한 증상은 서버의 모든 게이트웨이가 동시에 Enterprise Server와 통신을 멈추는 것입니다.

설치 만료 배너

Monnit은 iMonnit Enterprise 소프트웨어에 대해 Trial Enterprise 라이선스(60일 유효), Annual 센서 50개 라이선스, perpetual 센서 250개(이상) 라이선스를 제공합니다. 특정 기간 동안만 유효한 키를 적용했다면 라이선스를 갱신해야 합니다. 이 경우 iMonnit Enterprise가 게이트웨이로부터 데이터 수신을 멈추고, iMonnit Enterprise Portal 상단에 ‘This installation is expired, contact support to update subscription.’ 메시지가 배너로 보일 수 있습니다. 이 경우 유효한 키로 라이선스를 갱신해야 합니다.

참고: 레거시 버전(3.6.2.1 이하)의 Enterprise에서 센서 250개 라이선스를 가지고 있어도 이 메시지가 보이는 독특한 시나리오가 있습니다. 그 시나리오에 해당하면 해당 문서를 검토하십시오.

센서 한도

사용 가능한 iMonnit Enterprise 라이선스에는 서로 다른 센서 한도가 있습니다. 센서 한도에 도달하면 Enterprise Wireless Gateway Server 서비스는 계속 실행되지만 서버로 들어오는 통신을 수신하지 않습니다. 일반적으로 Enterprise 데이터베이스의 ExceptionLog 테이블에 예외가 표시됩니다. 가장 최근 로그를 검토하면 이 경우의 로그를 볼 수 있습니다.

데이터베이스 연결·크기·한도 확인

무료 버전의 SQL Express를 사용하는 경우, 이 문서에 따라 데이터베이스가 10GB 한도를 넘지 않았는지 확인해야 합니다. 이 문서로 유효한 데이터베이스 연결과 자격 증명도 확인할 수 있습니다.

Enterprise Wireless Gateway Server 서비스가 실행 중이고 라이선스가 유효함을 확인했다면, 해당 문서의 ‘iMonnit Enterprise 같은 로컬 서버와 Communication Check Utility 사용’ 섹션에 따라 Communication Check Utility로 테스트해야 합니다.

수행할 테스트는 두 가지입니다:

  1. iMonnit Enterprise를 실행하는 서버 자체에서 유틸리티를 실행해 테스트합니다.

테스트가 실패하지만 Enterprise Wireless Gateway Server 서비스가 실행 중이면, iMonnit Enterprise Portal의 구성된 포트(기본 3000)로 트래픽이 도달하는 것을 막는 PC 보안 설정을 분석해야 합니다.

  1. 같은 네트워크 세그먼트의 PC에서 테스트합니다.

같은 네트워크 세그먼트의 다른 PC에서 테스트하면 구성된 포트로 서버 통신을 막는 네트워크 구성을 식별하는 데 도움이 될 수 있습니다. 서버에서는 성공하지만 노드 PC에서 실패하면 네트워크 구성을 검토해야 합니다. 서버와 노드 PC 모두에서 성공하면 서브넷에 네트워크 구성 문제가 없을 가능성이 높습니다. 다만 노드 PC에서의 성공 테스트가 네트워크 구성에 유용한 지표이긴 하지만, 게이트웨이가 그 네트워크 세그먼트에서 Enterprise 서버에 도달할 수 있음을 확정하지는 않습니다. MAC 주소 필터링 같은 다른 보안 기능이 여전히 존재할 수 있으므로 좋은 지표일 뿐 확정은 아닙니다.

게이트웨이 검토

서버(가급적 노드 PC에서도)에서 Communication Check Utility 테스트를 성공적으로 수행했다면, Enterprise 서버가 구성되어 게이트웨이 통신을 성공적으로 수신하고 있음을 확인한 것입니다. 이 시점에 게이트웨이와 네트워크 구성을 검토해야 합니다.

게이트웨이 LED

게이트웨이 상태를 판단하려면 게이트웨이 LED를 검토합니다. 이 문서에서는 Ethernet Gateway 4 구성에 초점을 맞춥니다. 다른 게이트웨이 유형도 Enterprise와 사용할 수 있지만 Ethernet Gateway 4가 Enterprise에서 가장 흔합니다.

  • 하단 LED는 로컬 Ethernet 연결을 나타냅니다. 녹색 고정이면 게이트웨이가 로컬 네트워크에서 유효한 IP 주소를 가진 것입니다. 이것이 게이트웨이가 Enterprise Server에 도달할 네트워크 경로가 있음을 반드시 의미하지는 않지만, 게이트웨이가 로컬 네트워크에서 통신하고 있다는 좋은 신호입니다.

하단 LED가 빨간색이면 게이트웨이가 유효한 IP 주소를 얻지 못한 것입니다. 게이트웨이가 Enterprise Server에 도달하기 전에 이를 해결해야 합니다.

  • 두 번째 LED는 Enterprise 서버와의 연결을 나타냅니다. 녹색이면 서버와의 유효한 연결이 있는 것입니다. 유효한 서버 연결이 없으므로 이 LED가 빨간색일 가능성이 높습니다.

게이트웨이 IP 주소 Ping

게이트웨이 하단 LED가 녹색이고 게이트웨이 IP 주소를 안다면, Enterprise Server에서 그 IP 주소로 ping합니다. 정상 응답이 오면 Enterprise 서버에서 게이트웨이에 도달할 수 있는 네트워크 경로가 있는 것입니다.

ping이 unreachable을 반환하면 연결을 막는 것이 무엇인지 네트워크 구성을 검토해야 합니다.

게이트웨이의 로컬 HTTP 구성 페이지 접근

게이트웨이에 ping할 수 있으면, 게이트웨이의 로컬 HTTP 구성 페이지에 접근할 수 있는지 확인하십시오. 브라우저에 게이트웨이 IP 주소를 입력하면 됩니다. 게이트웨이 구성 페이지가 보이면 게이트웨이와 서버 간 HTTP 트래픽을 전송할 경로가 있음을 확인한 것입니다.

참고: HTTP 구성 페이지를 방문할 때, 이전에 활성화하지 않았다면 인터페이스가 비활성화되어 있거나 읽기 전용이라는 메시지가 보일 수 있습니다. 이 단계의 목적은 단순히 서버와 게이트웨이가 통신할 경로가 있는지 확인하는 것이지, 게이트웨이 구성을 편집하는 것이 아닙니다.

게이트웨이의 서버 호스트 주소 검토

게이트웨이의 로컬 HTTP 구성 페이지에 접근할 수 있으면 ‘Default Server Name/IP:’를 검토해 서버의 IP 또는 도메인 주소로 구성되어 있는지 확인합니다.

주소가 서버 주소로 올바르게 입력되어 있으면, 게이트웨이가 iMonnit Enterprise Portal로 데이터를 전달하도록 구성되고 Enterprise Server가 게이트웨이로부터 데이터를 받도록 구성된 것입니다. 둘 사이의 통신을 막는 네트워크 구성을 검토해야 합니다.

서버 구성이 양호하고 데이터를 수신하고 있음을 확인하고, 게이트웨이가 실제로 서버로 Point되어 있음을 확인했다면 Monnit 측 구성이 올바름을 확인한 것입니다. 기타 네트워크 및 보안 고려 사항은 적절한 담당자와 검토해야 합니다. 관련 문의는 언제든 support@monnit.com 으로 연락하십시오.

Sensor Profile별 구성 값

각 Sensor Profile에는 고유한 구성 값과 속성이 있습니다. 이 구성들은 센서가 제시하는 측정값에 자주 영향을 주는 센서 동작을 결정합니다. MINE 및 Serial MODBUS Gateway는 이 구성을 수정하는 기능을 제공하지만, Monnit은 이 센서 설정을 구성하는 데 필요한 설명을 명시적으로 제공하지 않습니다. 이는 Advanced Vibration, CO2 등 복잡한 구성 프로파일을 가진 센서 유형에서 특히 그렇습니다.

MINE의 센서 구성에 관한 참고

MINE의 한 기능은 모든 센서 유형에 정의된 ‘SensorEdit’ 메서드가 있다는 점입니다. 이 메서드의 매개변수는 iMonnit의 해당 센서 Edit 페이지에서 사용 가능한 매개변수와 동일합니다. 예를 들어 이 링크에서 MINE의 Temperature SensorEdit 메서드를 검토할 수 있습니다.

특히 이 매개변수들은 ‘Nullable’이어서 변경하려는 값만 전달하는 옵션도 제공합니다. 예를 들어 SensorEdit 메서드로 MINE 서버를 통해 센서에 Heartbeat 구성만 전달하면, 다른 매개변수는 기존 값을 유지하고 heartbeat 값만 변경됩니다. 이 메서드는 사용자를 위해 적절한 ‘dirty’ 플래그 표시도 처리합니다.

해당 메서드 안에 가드(guard)가 ‘이미’ 마련되어 있어야 합니다. 일반적으로 iMonnit UI에서 값을 구성할 수 있다면, MINE에서 메서드를 호출할 때 iMonnit과 동일한 값을 사용하면 동일한 결과가 나옵니다.

이것이 적용되지 않는 경우

앞서 언급했듯이 일부 센서 프로파일은 위 Temperature 센서보다 복잡합니다. MINE에서 원시 Sensor 객체를 수정해 이 구성을 편집할 수 있지만, 센서 동작을 방해할 수 있는 값 전달을 막는 가드가 없습니다. 이는 MINE에서 원시 Sensor 객체를 편집하는 경우와 Serial MODBUS Gateway로 센서를 구성하는 경우 모두에 적용됩니다. 이 구성들은 센서에 성공적으로 적용되려면 내부 정보가 필요하므로, 이 구성에 관한 세부 사항은 공개되지 않습니다. 잘못된 구성을 적용하면 센서가 동작 불능이 될 수 있습니다.

‘Object reference not set to an instance of an object.’ 오류가 보이는 경우

‘Object reference not set to an instance of an object.’ 오류는 데이터베이스 문제의 결과입니다. 이 오류를 해결하려면 데이터베이스 문제를 해결해야 합니다.

가능한 원인

  1. SQL 데이터베이스 크기
  2. 데이터베이스 연결 없음(권한, 사용자명/비밀번호, 네트워크 문제)

SQL 데이터베이스 크기 확인

가능성 높은 원인은 SQL 데이터베이스 크기입니다. 무료 버전의 SQL Express는 최대 10GB 데이터베이스를 허용합니다. 이 데이터베이스 한도에 도달하면 Enterprise 서버가 더 이상 데이터베이스에 쓸 수 없습니다. 정식 버전의 Microsoft SQL 응용 프로그램으로 업그레이드하거나 새 데이터베이스로 시작해야 합니다. 데이터베이스의 현재 데이터에는 접근할 수 있지만, iMonnit Enterprise 소프트웨어는 더 이상 데이터베이스에 접근/쓰기를 할 수 없습니다. 따라서 새 빈 데이터베이스를 생성해야 합니다. 나중에 데이터에 접근해야 한다면 먼저 기존 데이터베이스를 백업하는 것을 고려할 수 있습니다.

그런 다음 iMonnit Enterprise 제거 단계를 따를 수 있습니다. 제거 후 다시 설치하고 새 데이터베이스로 시작할 수 있습니다.

DB 연결 확인

위 단계를 거쳤는데도 문제가 지속되면 다음 중 하나에 문제가 있습니다:

  • SQL 사용자명/비밀번호 문제
  • Enterprise와 SQL 서버 간 네트워크 문제
  • 데이터베이스에 접근/쓰기하는 사용자의 권한

이 단계들로 데이터베이스 연결을 테스트하는 것이 유용할 수 있습니다. 문제가 지속되면 SQL 데이터베이스를 지원하는 담당자와 협력해 문제를 파악하십시오.
