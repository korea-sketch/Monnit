Monnit Wi-Fi(MoWi) 센서 설정 및 사용

Monnit MoWi 센서는 표준 무선 802.11 A, B, G 네트워크에 연결되어 iMonnit Online 포털로 데이터를 전송합니다. 게이트웨이가 선호되지 않는 상황에서 센서를 로컬 WiFi 네트워크에 연결할 수 있게 해 줍니다. 이 문서는 Monnit MoWi 센서를 WiFi 네트워크에 설정하는 과정을 안내합니다.

Link Mode(링크 모드)

Link Mode라는 용어는 Monnit 센서가 게이트웨이에 연결(또는 재연결)하려 할 때 수행하는 동작을 가리킵니다. 센서는 처음 연결을 시도할 때, 또는 게이트웨이와의 연결이 끊겨 게이트웨이를 찾아 재연결하려 할 때 Link Mode에 진입합니다.

센서에 영향을 주는 환경 요인

센서 동작에 영향을 줄 수 있는 몇 가지 환경 요인이 있습니다. 이 요인들은 센서의 전원 동작, 감지 동작, 무선 통신에 영향을 줄 수 있습니다. 이 문서는 이러한 요인에 관한 유용한 정보를 제공합니다. 고온과 저온 온도는 센서 동작에 영향을 주는 가장 흔한 요인 중 하나입니다. 온도는 폼팩터(외형)마다 다르게 영향을 줍니다.

Alta(2세대)와 Commercial(1세대) 기기의 차이

Monnit 기기는 두 세대로 제공됩니다. Alta 라인과 Commercial 라인은 서로 다른 무선 하드웨어로 동작합니다. 1세대 제품은 Alta 출시 이전에 Monnit이 개발했습니다. 두 세대는 고유한 특성과 동작을 가지므로 상호 호환되지 않습니다. 이 문서는 두 세대의 차이와 Alta 라인의 고유 기능을 설명합니다.

온도 센서 보정(Calibration)

Monnit 무선 온도 센서는 기본적으로 +/- 1°C(1.8°F) 정확도를 가지며, 센서가 동작하는 환경에 맞춰 보정하면 더 높은 정확도를 얻을 수 있습니다. 더 높은 정확도를 원하거나 온도 센서의 측정값이 부정확하다고 판단되면, iMonnit Online 포털에서 보정하거나 기본 보정으로 설정할 수 있습니다.

Sensor State vs. STS

MoWi 센서 또는 Ethernet 게이트웨이의 MAC 주소 조회 방법

이 문서는 MAC 주소를 가진 기기(MoWi 센서, Ethernet 게이트웨이 등)의 MAC 주소(및 기타 유용한 정보)를 조회하는 방법을 안내합니다. MAC 주소란? MAC 주소(Media Access Control address)는 네트워크상의 네트워크 인터페이스 컨트롤러(NIC)에 할당된 고유 식별자입니다.

한 번의 예정 하트비트에서 동일한 센서 메시지가 연속으로 중복 표시되는 경우

Monnit 프로토콜의 강점 중 하나는 센서가 송신과 수신을 모두 할 수 있다는 점입니다. 센서는 데이터를 전송한 후 센서 데이터가 수신되었다는 확인 응답을 게이트웨이로부터 듣기를 기다립니다. 일정 시간 내에 게이트웨이로부터 응답을 듣지 못하면 데이터를 재전송합니다.

H2S(황화수소 가스) 센서가 ‘Hardware Error!’로 표시되는 경우

H2S(황화수소 가스) 센서는 24시간의 초기화 기간을 가집니다. 이 기간 동안 센서는 동작 중이어야 하며 ‘Hardware Error!’를 나타내는 측정값을 제공할 수 있습니다. 이는 초기화 기간 동안 예상되는 동작입니다. 초기화 기간 H2S 센서는 긴 초기화 기간이 필요하다는 점에서 다소 독특합니다.

냉동고 제상 장치와 Monnit 센서

Monnit 센서 프로브나 센서 본체가 냉동고 또는 냉장고 제상 장치 근처에서 동작하면 센서 동작에 의도치 않은 영향이 있을 수 있습니다. 제상 장치는 흔히 전기 코일로 구성되므로 EMI 간섭의 형태로 센서 무선 통신에 영향을 줄 수 있습니다. 또한 제상 장치는 센서 측정 기록에 예기치 않은 고점과 저점을 유발할 수 있습니다.

Alta 센서 데이터 로깅

Alta 센서는 전원 손실과 무선 통신 손실 시에도 센서가 기록한 데이터가 유실되지 않도록 데이터 로깅 기능을 제공합니다. 현재 버전의 Alta 센서와 Alta 게이트웨이 모두 이 기능을 제공합니다. 이 문서는 센서 데이터 로깅의 기능을 자세히 설명합니다. 센서 데이터 로깅을 고려할 때 Link Mode를 이해하는 것도 중요합니다.

무선 안테나의 올바른 방향과 센서 위치

요약: 최적의 도달 거리와 성능을 위한 안테나 방향과 센서 위치 설정 방법 최적 도달 거리를 위한 올바른 안테나 방향과 센서 위치 설정. Monnit 무선 센서에서 최상의 성능을 얻으려면 올바른 안테나 방향과 센서 위치에 유의하는 것이 중요합니다. 안테나는 모두 같은 방향, 즉 센서에서 수직으로 나오도록 향해야 합니다.

48시간 넘게 Aware 상태에 있는 센서

요약 센서가 48시간 넘게 Aware 상태에 있으면, Aware 상태 기반의 Action(Advanced Notifications)이 더 이상 트리거되지 않습니다. 시스템은 센서가 모니터링하던 조건이 48시간 이내에 확인(acknowledge)되었을 것이라고 가정합니다.

무선 센서 네트워크에 맞는 무선 주파수 선택

요약 무선 센서 네트워크를 선택할 때, 특정 요구에 맞는 올바른 주파수를 고르려면 고려해야 할 몇 가지가 있습니다. Monnit의 무선 센서 네트워크는 현재 ISM 900MHz(902~928MHz) 대역과 868MHz, 433MHz 대역에서 동작합니다.

Monnit Standard 진동 센서와 ISO 10816

진동 모니터링은 유지보수 엔지니어가 전통적인 순회 점검 중 기계를 살펴보던 데에서 비롯되었습니다. 이는 기계가 평소와 다르게 진동하고 있다는 직관적 이해를 공식화하려는 시도입니다.

Monnit 전압 센서 응답 시간

Monnit이 제공하는 전압 계측기는 하드웨어에 약한 필터링이 구현되어 있어, 센서에 공급되는 실제 전압을 보고하는 데 약간의 지연이 생길 수 있습니다. 필터 시간 전압 센서에는 전압 급증으로부터 센서 회로가 과부하되지 않도록 보호하는 약한 필터가 있습니다. 이 필터는 5초의 지연을 가집니다.

고밀도 센서 네트워크 구성

요약 밀집 센서 네트워크를 위한 최적화 센서 통신 밀도: 경험칙으로, 최대 처리량으로 게이트웨이당 초당 1건(즉 분당 60건)의 센서 통신을 허용할 만큼의 대역폭을 두는 것을 선호합니다. 이론상 게이트웨이는 그보다 더 처리할 수 있지만, 센서가 완벽하게 무작위로 분포하는 경우는 드뭅니다.

습도 센서와 정확도 사양

Monnit 습도 센서는 제조 시 사양서에 명시된 범위 내에 있음을 확인받습니다. 환경 조건은 센서의 정확도에 영향을 줄 수 있습니다. 먼지나 화학물질 같은 것에 노출되면 Monnit 습도 센서의 정확도에 영향을 줄 수 있습니다. 따라서 센서가 제조사 사양을 벗어난 측정값을 보고한다면 이러한 요인을 고려하는 것이 중요합니다.

Monnit CO2 센서의 자동 보정(Auto Calibration) 기능

Monnit CO2 센서는 자동 보정(Auto Calibration) 기능을 제공하여, 자동 보정 기간 동안의 측정값을 기반으로 센서가 주기적으로 스스로 보정하도록 합니다. 작동 방식 이 예에서는 자동 보정 기간이 3일로 설정된 CO2 센서를 살펴봅니다.

다른 Monnit 센서와 비교하기

Monnit 센서를 보유 중이고 기기의 정확도가 우려된다면, 센서의 정확도를 검토할 수 있는 방법이 있습니다. 이는 특히 냉장 장치의 측정값이 Monnit 센서가 보고하는 값과 차이가 날 때 유용합니다. 이를 위해서는 프로브를 같은 영역에 놓을 수 있는, 동작 중인 여러 Monnit 온도 센서에 접근할 수 있어야 합니다.

구성 가능한 Window 필터를 지원하는 센서의 Window 필터링

Advanced Vibration Meter 등 일부 Monnit 센서는 센서 단에서 측정값을 필터링하는 Window 필터를 제공합니다. 이 필터를 용도에 맞게 조정하면 센서를 용도에 맞게 활용하는 데 도움이 됩니다.

열전대(Thermocouple) 센서가 ‘Invalid’로 표시되는 경우

Monnit 열전대(Thermocouple) 센서는 Readings 탭에 오류를 보고할 수 있습니다. 이 오류는 일반적으로 열전대 프로브와 센서 본체 간 접속 불량의 결과입니다. ‘Invalid’ 측정값이 흔히 ‘No Sensor Detected’와 함께 보일 수 있습니다.

Gateway Management 메시지 정의

Monnit 게이트웨이는 다양한 작업을 수행하며, 각 작업은 게이트웨이 History에 Gateway Management 항목으로 기록됩니다. Gateway Management는 이러한 각 작업에 대한 식별자입니다. 이 메시지의 의미를 이해하면 흔한 문제를 식별하는 데 유용합니다.

Serial MODBUS Gateway에서 ‘RX Timeout!’ 오류가 보이는 경우

처음 구성할 때(또는 설정 완료 후 가끔) Serial MODBUS Gateway에 명령이나 폴을 보낼 때 ‘RX Timeout!’ 오류가 보일 수 있습니다. 이는 통신 오류를 나타내며 일반적으로 구성 또는 배선 문제의 결과입니다. 이 문서는 이 문제의 해결 단계를 제공합니다.

배터리 백업이 있는 2G 또는 3G 셀룰러 게이트웨이에서 ‘Charge Fault’ 오류가 보이는 경우

Charge Fault 오류는 일반적으로 배터리 백업이 있는 셀룰러 게이트웨이의 전력 소비와 충전에 영향을 주는 물리적 조건을 나타냅니다. 이 문서는 이 문제와 가능한 해결 방법에 관한 정보를 제공합니다.

센서 메시지를 수신 즉시 전송하도록 게이트웨이 구성

기본 구성에서 게이트웨이 하트비트는 정해진 일정에 발생하도록 설정됩니다. 즉 게이트웨이는 센서 하트비트에 센서 통신을 수신하고, 게이트웨이 하트비트에 측정값을 Monnit 소프트웨어로 전달하기 위해 대기시킵니다. 이 동작을 변경해 게이트웨이가 센서 측정값을 수신하는 즉시 Monnit 소프트웨어로 전송하도록 만드는 방법이 있습니다.

USB 게이트웨이 또는 Wireless Sensor Adapter가 iMonnit 소프트웨어로 데이터를 보내지 않는 경우

예를 들어 Windows 구성 변경 후, USB 게이트웨이가 컴퓨터와 통신은 하지만 데이터가 iMonnit 소프트웨어에 도달하지 못하는 상황이 있을 수 있습니다. 게이트웨이가 PC에 인식되고 하드웨어에 문제가 없다면(설치 또는 드라이버 문제 해결 단계는 이 문서 참고), 다음 빠른 단계로 기본적인 문제를 해결할 수 있습니다.

게이트웨이와 같은 네트워크가 아닌 센서가 그 게이트웨이로 데이터를 보내는 경우(또는 내 네트워크 센서가 네트워크 외 게이트웨이로 데이터를 보내는 경우)

센서가 게이트웨이와 통신하도록 허가받는 수단은 Sensor List입니다. 게이트웨이가 센서를 Sensor List로 다운로드하면, 게이트웨이의 네트워크가 Reform되지 않는 한 그 센서 ID는 게이트웨이의 Sensor List에서 제거되지 않습니다.

iMonnit Online에서 별도 센서 네트워크로 센서를 게이트웨이에 지정

센서가 게이트웨이와 통신하도록 허가받는 수단은 센서 네트워크입니다. 즉 iMonnit Online에서 동일한 네트워크에 추가된 센서는 그 네트워크의 게이트웨이와 통신하도록 허가됩니다. 센서가 통신하도록 허가될 게이트웨이를 지정하려면 iMonnit Online에서 센서를 별도의 센서 네트워크에 할당해야 합니다.

iMonnit Online의 Charts 및 Min/Max/Average 표시

iMonnit Online의 Charts 기능은 개별 데이터 포인트로 생성된 선 그래프 또는 Min/Max/Avg 그래프 선이 있는 집계 데이터로 생성된 선 그래프를 표시할 수 있습니다. 이는 센서 데이터를 검토할 때 강력한 도구가 될 수 있습니다. 7일 미만의 날짜 선택은 개별 데이터 포인트로 생성된 그래프 선을 표시합니다.

iMonnit의 Action은 SMS 알림을 받도록 구성된 사용자에게 SMS로 알림을 전달할 수 있습니다. 각 통신사의 Email to SMS 시스템을 활용해 이메일을 SMS로 변환하는 무료 SMS 알림(External Delivery)이 있습니다. iMonnit 시스템에서 이 SMS 기능 사용에는 비용이 없습니다.

iMonnit Online에서 날짜 형식 변경

iMonnit Online은 사용자 선호에 맞출 수 있는 유연한 플랫폼입니다. 날짜 형식은 지역마다 다를 수 있으므로, 지역에 맞게 날짜 표시를 쉽게 구성할 수 있도록 했습니다. 날짜 형식 변경 파란색 탐색 바에서 Settings를 선택합니다. Account Settings 창 오른쪽 상단의 Preferences를 선택합니다.

Action이 트리거되는 동안 센서의 Action 일시 정지

센서가 이미 Action을 트리거한 상태에서 그 센서의 Action을 일시 정지하려는 시나리오가 있을 수 있습니다. Action을 트리거한 센서에서 이를 하면 Action이 자동으로 Disarm/Rearm되고, 센서가 더 이상 그 Action을 트리거해 알림을 보내지 않습니다.

사용자를 Action 수신자로 구성

이미 Action을 생성했는데 다른 사용자를 그 Action의 수신자로 추가해야 하는 경우가 있을 수 있습니다. 이 문서는 그 단계를 제공합니다. 단계 iMonnit Online 포털에 로그인합니다. 원하는 사용자가 생성되었고, 그 사용자의 Notification 세부 정보가 원하는 알림 유형으로 구성되었는지 확인합니다.

연결된 계정 접근 시 404 또는 453 오류

Linked Account를 활용하는 사용자가 있는 경우, 연결이 시작된 이후 연결된 계정 중 하나가 삭제되었다면 어떤 연결된 계정에 접근하려 할 때 404 또는 453 오류가 발생할 수 있습니다. 해결 이를 해결하려면 Monnit Support에 연락해 끊어진 링크를 제거해야 합니다.

iMonnit Online에서 센서 측정값에 메모(Notes) 추가

iMonnit Online 포털은 기록된 측정값에 메모를 추가하는 기능을 제공합니다. 이는 센서가 기록한 이벤트에 대한 설명을 추가하거나, 보정 수행을 문서화하거나, 기타 유사한 이벤트에 유용할 수 있습니다. 메모를 생성한 후 사용자는 특정 날짜 범위의 기록된 메모를 내보내는 Report를 보거나 생성할 수 있습니다.

온프레미스 소프트웨어

인터넷에 접속되지 않은 서버에서 iMonnit Enterprise를 활성화하는 경우

오프라인 활성화 iMonnit Enterprise 소프트웨어는 Enterprise Activation Key를 검증하기 위해 인터넷 연결을 사용합니다. iMonnit Enterprise를 설치하는 서버가 인터넷에 접속할 수 없으면, 서버에 인터넷 연결 없이도 활성화를 완료할 수 있는 오프라인 활성화 절차가 있습니다.

iMonnit Enterprise 서버에서 Enterprise Wireless Gateway Server 서비스 실행 확인

Enterprise Wireless Gateway Server는 iMonnit Enterprise 서버에서 Windows의 Services.msc 프로그램으로 실행되는 서비스입니다. 이 서버는 게이트웨이가 구성된 포트(기본 포트 3000)로 서버에 데이터를 전송할 때 들어오는 통신을 수신하는 역할을 합니다.

iMonnit Enterprise에 기기 추가 시 ‘could not be transferred to a new network’ 오류가 보이는 경우

‘[SensorID/GatewayID: xxxxxx] could not be transferred to a new network’ 오류는 iMonnit Enterprise 서버 버전 3.6.2.1 이하에 새 기기를 추가하려 할 때 나타날 수 있습니다. 이 오류는 몇 가지 문제 중 하나를 나타낼 수 있습니다.

iMonnit Enterprise 3.6.2.1 이하에서 ‘This installation is expired, contact support to update subscription.’가 보이는 경우

iMonnit Enterprise는 Annual 및 Perpetual 라이선스를 모두 제공합니다. Annual 라이선스의 경우 소프트웨어 활성화에 사용한 날짜로부터 1년 후 키가 만료됩니다.

Sensor Application ID 이해

Sensor Application ID 이해 Monnit은 각 센서 프로파일마다 식별 번호를 할당하며, 이는 모든 Monnit 플랫폼에서 그 센서 유형을 식별하는 데 사용됩니다. 이 식별자는 일반적으로 Application ID 또는 Profile ID라고 하며 10진수 형태입니다.

인터넷에 연결되지 않은 상태에서 iMonnit Enterprise 서버에 기기 추가

iMonnit Enterprise는 기기를 Enterprise 포털/데이터베이스에 추가할 때 기본적으로 인터넷 연결을 활용해 기기 메타데이터를 자동으로 다운로드합니다.

iMonnit Express 4 서비스 시작

iMonnit Express 4 소프트웨어는 웹 서버를 운영하며 Windows 서비스를 기반으로 통신을 수신합니다. 서버와 함께 실행되는 소형 웹 서버는 ‘localhost’ 주소로 접근할 수 있는 Kestrel 웹 서버입니다. 이 서비스는 또한 TCP 및 UDP 포트 3000에서 들어오는 게이트웨이 트래픽을 수신합니다.

호스트 컴퓨터가 오프라인인 상태에서 iMonnit Express에 센서를 추가해야 하는 경우

iMonnit Express 4.0.1(2020년 3월 출시)은 호스트 PC에 인터넷 연결 없이 기기를 추가하는 기능을 제공합니다. 일반적으로 기기를 추가할 때 iMonnit Express는 인터넷에 접속해 센서 메타데이터(센서 유형, 배터리 유형, 펌웨어 버전 등)를 다운로드하여 iMonnit Express 데이터베이스로 전달합니다.

iMonnit Express 4.0 사이트를 설치·접근한 후 보안 오류가 보이는 경우

iMonnit Express 4.0은 PC에 로컬로 호스팅되는 웹사이트입니다. 따라서 Express 설치 시 포함되는 보안 인증서(Security Certificate)가 없습니다. 그 결과 웹사이트 접근 시 보안 오류가 보일 수 있습니다. 실제 보안 위험은 없지만, 브라우저가 보안 인증서가 없는 사이트를 보안 문제로 해석합니다.

iMonnit Express 4와 함께 사용하기 위한 Ethernet 게이트웨이 구성

iMonnit Express 4는 iMonnit Online의 대안으로 Monnit 소프트웨어를 로컬에서 운영하는 iMonnit Express 소프트웨어의 현재 버전입니다. Express는 인터넷 연결 없이 독립형 Monnit 서버로 실행됩니다.

업그레이드 전 iMonnit Enterprise 백업 생성

새 버전으로 업그레이드하기 전에 iMonnit Enterprise 설치를 백업하는 것이 중요합니다. 이 문서는 그 단계를 안내하는 자료를 제공합니다. 단계 요약 사이트 파일 백업을 생성합니다. Enterprise 데이터베이스 백업을 생성합니다. Activation Key를 반드시 기록해 둡니다.

iMonnit Enterprise 설치 체크리스트

iMonnit Enterprise 사이트 로딩 시 오류가 보이면 다음을 확인하십시오. 설치 전에 이 항목들을 검토하는 것도 좋습니다. Configuration Utility를 관리자 권한으로 실행했는지 확인합니다(마우스 오른쪽 버튼 클릭 > Run as Administrator).

iMonnit Express 4에서 새 빈 데이터베이스 생성

iMonnit Express 4의 데이터베이스에 문제가 생겨 다시 만들어야 하는 경우가 있을 수 있습니다. 이 문서는 iMonnit Express 4에서 새 빈 데이터베이스를 생성하는 방법을 안내합니다. 단계 요약 Monnit 또는 리셀러로부터 현재 iMonnit Express 4.0 Activation Key를 구매/확보합니다.

iMonnit Express 4.0 오프라인 활성화

iMonnit Express 4.0은 인터넷 연결 없이 활성화하고 기기를 추가하는 기능을 제공합니다. 이 문서는 이 작업에 필요한 정보를 제공합니다. 참고: 이는 Express 버전 4.0.1 이상에 해당합니다.

iMonnit Enterprise - ‘Error 500.19 - Internal Server Error’

요약 iMonnit Enterprise를 설치한 후 누락된 Windows Features로 인해 다음 오류가 발생할 수 있습니다. 해결책: 이는 일반적으로 서버에 ASP.NET(또는 기타 필수 기능)이 활성화되지 않은 결과입니다.

iMonnit Express 4.0 설치·활성화 후 ERR_TOO_MANY_REDIRECTS(또는 유사) 오류가 보이는 경우

iMonnit Express 4.0 소프트웨어를 설치하고 활성화하려 할 때 ERR_TOO_MANY_REDIRECTS 오류가 보일 수 있습니다. 이는 이전 버전의 iMonnit Express용 Activation Key를 사용한 결과일 가능성이 높습니다.

Monnit MINE Java Sample Application GUI 시작하기

Monnit MINE은 Monnit 게이트웨이·센서를 타사 플랫폼과 연동할 수 있게 하는 독립형 서버입니다. Monnit 기기는 독자 프로토콜로 동작하므로 Monnit 시스템과 직접 연동하려면 Monnit MINE이 필요합니다. MINE은 Monnit 하드웨어의 통신을 수신하고 다른 플랫폼과 연동할 수 있게 하는 ‘미들웨어’ 서버입니다.

iMonnit 계정 계층(Account Hierarchy)

iMonnit 소프트웨어는 최상위 계정이 계정, 사용자, 기기, 네트워크, 서브 계정(중첩 계정 포함)을 생성·관리·접근할 수 있는 계정 계층을 구현합니다. 이 계층을 이해하는 것은 성공적인 구축 관리에 중요할 수 있습니다. 특히 관리할 서브 계정과 중첩 계정이 있을 때 그렇습니다.

iMonnit Express 4.0 서비스가 시작되지 않는 경우

iMonnit Express 4 소프트웨어를 설치한 후 iMonnit Express 서비스를 시작하려 할 때 ‘Error 1064: An exception occurred in the service when handling the control request’ 오류가 보일 수 있습니다.

Monnit MINE을 AWS와 연동하기

이것은 Monnit MINE을 AWS와 연동하기 위한 간단한 ‘How-To’ 가이드입니다. 고지: 이 절차는 AWS에 대한 지식이 있는 소프트웨어 개발자가 완료해야 합니다.

Enterprise 완전 제거 방법

요약 컴퓨터에서 Enterprise 설치를 완전히 제거하려면 다음 단계를 따르십시오. Enterprise 설치 중 문제가 계속될 때 유용합니다. 전체 재설치를 의도하는 경우에만 이 단계를 따르십시오. 서비스 중지 Windows 키를 누르고 ‘Services’를 입력합니다.

iMonnit Enterprise SQL 데이터베이스의 유지보수 요구 사항

시간이 지나면 iMonnit Enterprise SQL 데이터베이스에는 유지보수가 필요합니다. 그 유지보수의 정확한 적용은 일반적으로 데이터베이스가 어떻게 사용되는지에 따라 다릅니다. 한 데이터베이스에 최적인 것이 다른 데이터베이스에는 가장 최적이 아닐 수 있습니다. 다만 데이터베이스 유지에 할 수 있는 일반적인 조언이 있습니다.

Monnit Enterprise Configuration Utility 외부에서 SQL 데이터베이스 연결 테스트

SQL 플랫폼은 데이터베이스 연결을 테스트하는 방법을 제공합니다. 데이터베이스 연결에 문제가 있으면 Enterprise 설치 프로그램 외부에서 테스트하여 자격 증명이 올바른지 확인할 수 있습니다. 테스트 완료는 아래 단계를 참고하십시오.

Express 완전 제거 방법

단계 서비스 삭제 적절한 권한이 있는 사용자로 Windows에서 Services.msc 프로그램을 실행합니다(Windows 검색에서 ‘services’ 검색). ‘iMonnit Express’ 서비스를 찾아 마우스 오른쪽 버튼으로 클릭하고 Stop을 선택합니다.

iMonnit Enterprise 설치가 연결된 SQL 데이터베이스의 자격 증명 변경

Enterprise 설치가 데이터를 기록하는 SQL 데이터베이스의 자격 증명을 변경해야 하는 경우가 있을 수 있습니다. 일반적으로 SQL 사용자 자격 증명이 변경되어 Enterprise에서 업데이트해야 하거나, Enterprise가 다른 데이터베이스로 데이터를 보내도록 지정해야 하는 경우입니다.

Enterprise 4로 업그레이드 시 고려 사항

지원 버전 Enterprise 4로 업그레이드하려면 버전 3.5.0.0 이상이 필요합니다. 3.5.0.0 이전 버전은 iMonnit Enterprise 4.0으로 성공적으로 업데이트할 수 없습니다. 기존 사이트 백업 Enterprise 4로 업그레이드하기 전에 이 문서에 따라 기존 Enterprise 설치를 백업해야 합니다.

iMonnit Enterprise 갱신 키 입력

iMonnit Enterprise를 센서 10개 Trial 키 또는 센서 50개(Annual) 키로 활성화했다면, 소프트웨어를 재활성화하기 위해 설치 프로그램에 갱신 키를 입력해야 합니다. 이 문서가 그 방법을 안내합니다.

게이트웨이가 iMonnit Enterprise와 예기치 않게 통신을 멈추는 경우

모든 게이트웨이가 갑자기 iMonnit Enterprise Portal로 데이터 전송을 멈추면, 이 문서의 정보가 문제 식별에 도움이 됩니다. iMonnit Enterprise Portal 검토 가장 먼저 검토할 구성 요소는 iMonnit Enterprise Portal입니다.

Sensor Profile별 구성 값

각 Sensor Profile에는 고유한 구성 값과 속성이 있습니다. 이 구성들은 센서가 제시하는 측정값에 자주 영향을 주는 센서 동작을 결정합니다. MINE 및 Serial MODBUS Gateway는 이 구성을 수정하는 기능을 제공하지만, Monnit은 이 센서 설정을 구성하는 데 필요한 설명을 명시적으로 제공하지 않습니다.

‘Object reference not set to an instance of an object.’ 오류가 보이는 경우

‘Object reference not set to an instance of an object.’ 오류는 데이터베이스 문제의 결과입니다. 이 오류를 해결하려면 데이터베이스 문제를 해결해야 합니다.

애드온 기기

센서의 하드웨어 오류(Hardware Failure)

코인셀 배터리로 동작하는 센서의 경우, 이 측정값은 일반적으로 배터리가 잘못 장착되어 발생합니다. 배터리를 하우징 안으로 더 밀어 넣어야 하며, 배터리 위치가 센서의 데이터 측정 능력을 방해하고 있는 것입니다. 배터리가 내부 회로 기판 가장자리의 원형 포트에 닿아 있을 가능성이 높습니다.

게이트웨이 및 Add-On 기기의 Poll Interval

Poll Interval은 Control Unit, Local Alert 같은 Add-On 기기의 원하는 동작에 중요한 구성입니다. Polling 과정은 Add-On 기기가 원하는 응답 시간으로 명령이나 입력에 응답할 수 있게 합니다. 고려할 Polling 이벤트는 두 가지입니다.

Control Unit의 Default State

Monnit Control Unit에는 각 Relay의 정상 동작 상태를 결정하는 Default State 구성이 있습니다. 각 Relay는 Off 또는 On의 Default State를 갖도록 독립적으로 구성할 수 있습니다. Default State란?

Control Unit Relay 테스트

Control Unit Relay의 열림/닫힘에 문제가 의심되면 멀티미터로 릴레이를 테스트할 수 있습니다. 이 문서는 그 방법에 관한 정보를 제공합니다. LED 표시 확인 각 Control Unit에는 물리적 LED 4개(Relay 1, Relay 2, Radio, Power)가 있습니다.

기기 손상

센서 또는 게이트웨이를 수리를 위해 Monnit으로 보내기

Monnit은 의도대로 동작하지 않는 제품에 대해 수리 서비스를 제공합니다. Monnit 1년 보증 기간 내 제품의 경우, 손상이나 오용이 감지되지 않으면 Monnit이 수리 또는 교체(동일/유사품 및/또는 재생/신품)를 부담합니다.

센서가 Improper Battery Placement를 보고하는 경우

Monnit 센서는 배터리가 올바르게 삽입되지 않았거나, 센서 회로에 단락을 일으키는 하드웨어 문제가 있으면 Improper Battery Placement 오류를 보고합니다. 이 오류가 보이면 고려·확인할 사항이 몇 가지 있습니다.

센서가 iMonnit Online에서 ‘Short Detected’ 오류를 표시하는 경우

Short Detected 오류는 센서 하드웨어가 회로에서 단락을 감지했음을 나타냅니다. 이 문서는 이 오류의 가능한 원인과 해결책을 안내합니다.

센서가 No Sensor Detected를 보고하는 경우

일부 센서는 센서 프로브에서 측정값을 가져올 수 없을 때 ‘No Sensor Detected’ 오류를 보고합니다. Temperature, Humidity, Thermocouple 등 일부 센서 유형이 이 오류를 보일 수 있습니다.
