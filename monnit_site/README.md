	# Monnit Korea 사이트 — 파일 구조 안내

기존 단일 HTML(약 6MB)을 역할별로 분리했습니다. 이제 무엇을 고치든 해당 파일 하나만 열면 됩니다.

## 폴더 구성

```
monnit_site/
├─ index.html   (66 KB)   ← 화면 구조·문구·섹션. 텍스트 수정은 대부분 여기
├─ style.css    (133 KB)  ← 디자인·색상·애니메이션·반응형
├─ app.js       (180 KB)  ← 동작 로직(필터·탭·폼·렌더링)
├─ data.js      (1.5 MB)  ← 지식베이스 215개 문서 + 가이드 콘텐츠
└─ images/      (45개)    ← 로고·인물·배경 등 분리된 이미지
```

## 무엇을 어디서 고치나

| 하고 싶은 일 | 열 파일 |
|---|---|
| Why Monnit 문구, 섹션 제목 등 화면 텍스트 | `index.html` |
| 색·여백·폰트·버튼 모양·애니메이션 | `style.css` |
| 필터/탭 동작, 폼 처리, 데이터 렌더링 방식 | `app.js` |
| 도움말 문서·가이드 내용 추가/수정 | `data.js` |
| 사진 교체 | `images/` 폴더의 해당 파일 교체(같은 이름) 또는 새 파일 후 src 변경 |

## 중요 — 로드 순서

`index.html` 맨 아래에서 **`data.js` → `app.js`** 순서로 불러옵니다.
이 순서를 바꾸지 마세요. (app.js가 data.js의 데이터를 사용합니다.)

## 실행 방법

`index.html`을 더블클릭하면 브라우저에서 그대로 열립니다.
배포 시에는 이 폴더 전체(images 포함)를 통째로 올리면 됩니다.

## 수정 의뢰 팁

Claude에게 수정을 맡길 때:
- 이미지가 빠진 가벼운 `index.html`만 올려도 텍스트·구조 수정은 충분합니다.
- 디자인 수정이면 `style.css`, 동작 수정이면 `app.js`만 올리면 됩니다.
- "이 문장을 이렇게 바꿔줘"처럼 위치+내용을 지정하면 가장 안정적입니다.
